# Notice

This project is a content workflow / skill kit. It does not automate publishing, platform login, comments, direct messages, or private data collection.

Some workflow design ideas, skill decomposition habits, and Chinese self-media quality review thinking were inspired by `dbskill`. This project keeps its own scope:

```text
hotspot -> mother topic -> product communication -> script -> picture-in-picture prompt/assets -> quality review -> platform package -> final HTML delivery
```

External references included under `外部资料/` are research material only and are not runtime dependencies.

## Release Boundary

```text
public_release candidate: generated package for review and testing.
GitHub release: only valid after user-approved commit, tag, remote, push, and release notes publication.
```

## Method References

Release notes and changelog structure are informed by mature open-source practices such as semantic versioning, Keep a Changelog style history, GitHub Releases, and GitHub community health files.
