## 已确认

### R5-C01 账号级视觉身份

每个账号有独立、轻量的视觉身份；视觉一致性跟随账号，而不是跟随某一篇内容。

### R5-C02 身份是语法，不是模板复制

身份约束证据感、语气、层级、禁忌和栏目表达，不要求每张图同色、同构图或强塞 Logo。

### R5-C03 图片数量不归账号固定

账号身份只影响“如何表达”。单篇图片仍由 R3 的 `visual_need_analysis` 决定，允许 `0..N`。

### R5-C04 工作流共用、账号独有

雷达引擎、事件模型和视觉规划器是工作流共用能力；账号策略、词库、记忆、视觉身份和栏目模板必须账号隔离。

当前编译口径：新建或启用的视觉身份与栏目模板使用 v0.2 account-bound 合同，同时携带并校验 `account_identity_id`、`account_technical_slug`；历史 v0.1 只用于 replay，不能作为当前跨账号启用证据。

### R5-C05 二手车为第一优先级

账号策略优先发现和筛选二手车直接相关事件，不以泛汽车热度替代。

### R5-C06 新车外溢的触发阈值

当本轮不足 3 条“事实可核验且二手车直接相关”的候选时，才允许开启新车外溢池。

### R5-C07 新车必须证明影响链路

新车候选必须说明残值、置换、车源、库存/现金流、二手新能源检测/流通或政策传导等影响；没有证明不得入选。

### R5-C08 三类受众，但一条候选有主次

账号服务买车消费者、卖车用户与车商经营者；候选必须明确 `audience_primary`，可补次受众。

### R5-C09 三态与风险分开

来源/事实、平台传播、风险、账号匹配和排序分分别记录；任何内部排序分都不是事实置信度。

### R5-C10 自由探索、结果反馈的关键词扩展（以 D12 / D13 修订为准）

账号提供种子词、母题、受众边界与禁区；雷达引擎可直接使用关联搜索、事件实体和共现词扩展探索查询，不设逐词人工批准门禁。扩词是否继续升权，由它实际辅助发现有效候选、辅助被选择或持续产生误报的反馈决定；账号硬禁区、合规边界和用户明确封禁仍然优先。该条已由 R5-D12 / D13 取代早期“人工审核后才可使用”的草图口径。

## 本轮已确认的底层雷达模型

### R5-D10 四层对象模型

已认可采用：

```text
signal -> event -> candidate -> topic
```

同一事件的多篇报道、热词和平台观察先归并为 `event`，再按账号产生候选。这样“帅车”这类事件既可追溯，也不会因不同标题重复占据候选池。

### R5-D11 趋势声明要有两次快照

已认可：第一次只能写“新发现”；只有同源、同口径的至少两次可比快照后，才能标为 `rising`、`sustained` 或 `cooling`。缺数据时写 `not_comparable`，不编造“热度正在上涨”。

### R5-D12 人工批准白名单（已废止）

不采用“扩词逐词人工批准后才可使用”的模式。

### R5-D13 自由扩词与选择反馈（已确认）

关联搜索、事件实体、共现词等可直接进入受账号边界约束的探索查询。每个扩词和组合记录运行、辅助发现、形成候选、辅助被选择与辅助被拒绝计数；被选择的有效候选越多，后续权重越高，持续带来误报则自动降权。只有账号硬禁区、合规边界或用户明确封禁才进入 `blocked`。

## 编译前必须闭合的合同

### R5-C11 账号档案与字段词典升级

必须定义并同步 `account_identity_id`、`account_technical_slug`、`radar_policy_ref`、`audience_priority`、`topic_priority_rules`、`used_car_priority_mode`、`new_car_spillover_threshold`、`new_car_spillover_proof_types`、`topic_exclusions`、`query_lexicon_ref`、`hotspot_memory_ref`、`visual_identity_ref`、`column_visual_template_refs`、`account_snapshot_ref`、`account_snapshot_status` 和 `high_risk_topic_policy`。其中展示名、私有目录键和技术 slug 不可混用。

### R5-C12 私有记忆边界

真实账号词库、热点记忆、人工监看理由与运行信号只在 `accounts/{账号名}/` 私有区保存；公开 Git、样例和发布包只保留脱敏合同与 fixture。

### R5-C13 人工监看诚实性

`manual_watchlist` 可以保证已知重大事件不遗漏，但必须保留加入/复核理由，且不得计入自动发现率或通过无品牌词发现 fixture。

### R5-C14 排序可解释

排序必须输出 `ranking_components` 与原始观测；没有来源、基线或证据的数字只能显示为不可比较，不能包装为“机会分”。

### R5-C15 事件归并与演化

`event_fingerprint` 至少由主体、动作、时间窗、地点和业务链路组成；新增报道追加证据和状态，不覆盖既有事实记录。

### R5-C16 高风险选题门禁

高风险事件可以进入核验队列，但不得绕过事实状态、风险提示和人类选题确认直接生成口播文案。

### R5-C17 词库选择反馈账本

查询批次、命中、有效候选、误报、漏报、人工监看占比、验收事件覆盖和扩词的辅助选择计数需写入可复查账本；它用于自动形成账号偏好，不替代内容价值判断。

### R5-C18 账号切换隔离

切换账号必须重读该账号策略、词库、记忆、视觉身份和栏目模板；不得沿用前一账号的候选排序或视觉资产。

### R5-C19 正反 fixture

至少覆盖无品牌词发现、无二手车关联的新车拒绝、阈值外溢、同事件归并、单快照趋势诚实、人工监看标记和误报词拒绝。

### R5-C20 编译与真实回归边界

产品确认后先编译字段合同、对象合同、fixture 和 checker；真实账号回归仅在私有生产区进行，不自动登录平台、不自动发布，不因一次回归结果把真实数量写回通用合同。

### R5-C21 账号启动检查与最小补问（已确认）

任何热点、选题或内容 Skill 在读取账号后，先执行账号启动检查，而不是把不完整的账号散文档案直接交给下游猜测。检查输出必须区分 `missing_fields`、`blocking_fields`、`non_blocking_fields`、`account_snapshot_ref`、`account_snapshot_status` 与 `next_skill`；结果只可为 `account_ready`、`account_needs_input`、`account_policy_incomplete` 或 `account_blocked`。

只询问当前任务真正需要、且尚未确认的字段：每轮最多 3 个自然语言问题，不展示字段名表单。不得从过往文章或前一账号静默推断受众优先级、平台、时长或风险口径；若用账号档案中的暂定值启动提问，必须标为 `inferred_from_profile` 并要求用户确认。用户回答后生成或更新版本化的 session 账号快照；下游只读取该快照，不重新自由解释原始档案。

热点雷达的最小阻断组是：账号雷达策略 / 词库引用可用、三类受众的当前优先级，以及高风险事件的处理口径。视觉身份缺失不阻断热点发现；它只在需要视觉规划或最终视觉交付时变成相关阻断项。账号选择切换时不得复用上一个账号的快照、词库、候选或风险设定。

对于车商资金、代理履约、维权、事故等高风险题材，默认口径是 `verify_mechanism_only`：先交叉核验事实，讲行业机制和受影响链路，不对具体主体直接下结论。没有用户明确确认的可核验来源时，启动检查必须停在 `account_needs_input` 或 `account_blocked`，不能自动进入文案。

### R5-C22 账号启动检查的验收口径（已确认）

至少覆盖：缺少平台 / 时长时只问该组；三类受众都有但无优先级时只问优先级；纯热点任务缺视觉身份时继续；视觉交付任务缺视觉身份时按需补问；高风险候选无口径时只进入核验 / 机制拆解；切换账号后不能读取前账号快照；用户拒绝补全或超过本轮可答范围时保持 `account_needs_input`，不悄悄用默认值继续。

### R5-C23 跨账号技术身份绑定与自动校验（已确认，R5-H6 已编译）

多账号不是可选扩展，而是本工作流的基础合同。每个账号必须同时具有不变的 `account_identity_id`、ASCII 机器标识 `account_technical_slug`、人类可读的 `account_display_name` 与私有目录键；中文展示名和目录名可以相同，但都不能替代技术 slug。账号档案是唯一绑定真源；雷达策略、词库、热点记忆、视觉身份、栏目模板及 session 快照都必须携带同一 `account_identity_id` / `account_technical_slug`，并可追溯到当前绑定版本与 digest。

启动检查必须从用户选择的账号解析该绑定，再验证全部引用仍位于所选账号私有根目录、引用对象的技术 slug 一致、快照的绑定 digest 未过期。缺少绑定、slug 不一致、目录越界、旧快照串号或同展示名但不同技术身份时，结果必须为 `account_identity_inconsistent`，直接阻断下游；它不是向用户补问的内容，也不得从中文名、目录名或前一账号静默猜测 / 回退。旧账号需显式迁移并写出映射证据后才可继续。

验收 fixture 至少覆盖：同展示名不同技术身份、目录与档案 slug 不符、策略 / 词库 / 视觉身份各自错绑、旧快照错绑、私有根目录逃逸、显式迁移成功和正常切换账号。R5-H6 已重新闭合字段词典、Schema / runtime、Skill / CONTRACT、fixture 与 checker；一份私有账号已完成显式迁移和真实启动回归，不能用 H5 的旧快照隔离证据替代，也不能把该回归外推为已执行真实热点采集或发布。

## R8 实测反哺的待确认合同

### R5-C24 账号内容语言进入 session snapshot（待确认）

账号档案和任务快照必须显式携带 BCP 47 `primary_content_language`。内容生产与平台
包装不得从聊天、账号中文名、历史文章或执行者常识静默推断语言。current 内容 /
平台 snapshot 的主语言为必填。

### R5-C25 平台语言覆盖和账号表达约束（待确认）

账号可用 `platform_language_overrides` 为具体平台覆盖主语言；没有覆盖时继承
primary。任务快照同时携带可直接执行的 `account_voice_constraints` 以及
`style_guide_ref + digest`，避免下游只知道发布平台却不知道账号如何表达。
voice 至少一条；style ref / digest 成对必填并绑定 current 账号文件。

### R5-C26 缺语言 / voice 的任务相关补问（待确认）

语言或账号表达约束对当前内容生产 / 平台包装必需而未确认时，启动检查保持
`account_needs_input`，一轮最多三个自然语言问题。纯热点发现如果暂不生产内容，
可将其标为后续阶段阻断项，不因此阻断事实发现。

### R5-C27 首次观察和趋势状态分离（待确认）

热点对象使用 `observation_status=first_observation|repeated_observation` 与
`trend_status=not_comparable|rising|sustained|cooling` 两轴。第一次固定
`first_observation + not_comparable`，人类标签为“首次发现，尚无趋势结论”。

### R5-C28 current 废止含混的 new_observation（待确认）

`new_observation` 只用于历史 replay，不再作为 current `trend_status`。这不改变
“至少两次同口径快照后才可声明趋势”的既有规则，只消除交付表达歧义。

### R5-C29 版本化迁移与账号隔离（待确认）

现有账号采用显式迁移生成新 snapshot；语言、voice、style digest 必须与同一
`account_identity_id / account_technical_slug` 绑定。切换账号不得继承前一账号的
语言 override 或表达约束。
