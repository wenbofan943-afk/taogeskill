# R1 字段级输入输出矩阵

> 归档状态：historical_audit_only；2026-07-18 移出当前产品热路径，不得作为 current 合同真源。

> 状态：已确认并已编译引用  
> 所属路线：R1 方法论没有编译成执行合同  
> 解决缺口：把 skill 合同里的段落级输入输出，收敛成字段级接线表。  
> 边界：本文件不替代 `交接物字段词典.md`；字段最终真源仍需回写字段词典和对应 CONTRACT。当前不改 `skills/*/SKILL.md`。

---

## 1. 为什么要有字段矩阵

R1 之前已经说明“每个 skill 接什么、产什么”，但还不够细。

如果没有字段矩阵，编译时仍会靠 agent 读段落理解：

```text
这个字段是不是必需？
这个状态值是不是通过？
缺字段时回到哪个节点？
下游到底读哪个文件里的哪个字段？
```

成熟 workflow 通常会把任务定义收敛成 schema / asset contract。  
本项目不引入重型 schema 系统，但 R1 至少要有一张人和 AI 都能读懂的字段级接线表。

---

## 2. 字段分级

R1 字段分四类：

| 类型 | 含义 | 缺失处理 |
|---|---|---|
| `required_input` | 当前 skill 必须读取的字段 | 缺失则不运行，回上游补齐 |
| `required_output` | 当前 skill 必须产出的字段 | 缺失则不能进入下游 |
| `routing_status` | 决定自动推进 / 人类门禁 / 返工的状态字段 | 缺失则阻断 |
| `trace_field` | 用于追溯、成熟度和断链检查 | 缺失则至少 WARN，关键链路可 BLOCKER |

字段真源优先级：

```text
session manifest / session artifact
> 交接物字段词典.md
> skills/{skill}/CONTRACT.md
> docs/product 草案
```

聊天上下文不能成为字段真源。

## 3. 全链路必贯穿字段

以下字段从上游到下游必须连续：

| 字段 | 起点 | 终点 | 断链后果 |
|---|---|---|---|
| `session_id` | manifest | final_delivery | BLOCKER |
| `account` / `account_id` | account_profile | final_delivery | BLOCKER |
| `product_profile_id` 或 `campaign_profile_id` | product/campaign profile | content_delivery_record | BLOCKER |
| `research_run_id` / `source_research_run_id` | research_run_record | content_delivery_record / final_delivery | BLOCKER |
| `topic_id` | topic_card | content_delivery_record | BLOCKER |
| `brief_id` | content_brief | content_delivery_record | BLOCKER |
| `draft_id` | draft | content_delivery_record | BLOCKER |
| `visual_plan_id` | visual_plan | content_delivery_record | WARN，若最终 HTML 需要画中画则 BLOCKER |
| `quality_review_id` | quality_review | content_delivery_record | BLOCKER |
| `platform_package_id` | platform_package | content_delivery_record | BLOCKER |
| `delivery_record_id` | content_delivery_record | final_delivery | BLOCKER |
| `final_delivery_id` | final_delivery | workflow_session_record | BLOCKER |
| `contract_set_version` | R1 合同组 | manifest / trace | R1 编译后新样本必填 |

## 4. Skill 输入输出矩阵

### 4.1 propagation-router

| 类型 | 字段 |
|---|---|
| required_input | `user_intent`、`project_stage`、`workflow_usage_state`、`current_stage`、`current_artifact` |
| optional_input | `session_id`、`account`、`latest_manifest_path`、`latest_trace_path` |
| required_output | `router_decision`、`next_skill`、`reason`、`human_prompt`、`auto_next_action`、`task_after_navigation` |
| routing_status | `route_status` |
| downstream | 对应专项 skill 或 human_gate |
| missing_policy | 状态不清先读 `工作流状态记录.md` 和 manifest；账号不清停在账号确认 |

### 4.2 hotspot-topic-research

| 类型 | 字段 |
|---|---|
| required_input | `account_profile_id`、`account_profile_status`、`account_profile_confirmed_for_session`、`product_profile_id` 或 `campaign_profile_id` |
| required_output | `research_run_id`、`topic_id`、`topic_title`、`topic_angle`、`source_research_run_id`、`hotspot_time_window`、`hotspot_freshness_status`、`content_position`、`bridge_status`、`topic_status` |
| routing_status | `topic_status` |
| pass_status | `topic_selected_for_brief` 由用户选题后形成 |
| downstream | `content-brief-compiler` |
| missing_policy | 账号未确认回账号确认；产品对象缺失回 product_profile；时效缺失不得叫热点 |

### 4.3 content-brief-compiler

| 类型 | 字段 |
|---|---|

## 5. 状态枚举接线

R1 编译时最小必须识别这些通过状态：

| artifact | 状态字段 | 通过值 | 下游 |
|---|---|---|---|
| `topic_card` | `topic_status` | `topic_selected_for_brief` | `content-brief-compiler` |
| `content_brief` | `brief_status` | `brief_pass` | `copywriting-draft-writer` |
| `draft` | `draft_status` | `draft_created` | `talking-head-image-pip` |
| `visual_plan` | `visual_plan_status` | `visual_plan_pass` | `copywriting-quality-review` |
| `quality_review` | `review_status` | `review_pass` | `platform-packaging-adapter` |
| `platform_package` | `package_status` | `package_pass` | `content_delivery_record` |
| `content_delivery_record` | `delivery_status` | `delivery_ready` | `final-delivery-builder` |
| `final_delivery` | `final_delivery_status` | `html_ready` / `bundle_ready` / `standalone_ready` | `human_final_review` |

如果状态字段缺失，下游不得凭正文猜测通过。

## 6. 缺字段恢复规则

| 缺失位置 | 恢复动作 |
|---|---|
| `account_profile` 必需字段缺失 | 回账号档案 P0 检查 |
| `product_profile_id / campaign_profile_id` 缺失 | 回产品 / 活动对象确认 |
| `source_research_run_id` 缺失 | 回调研运行记录，不继续内容链 |
| `topic_status` 缺失 | 回 topic_card，不生成 Brief |
| `brief_status` 缺失 | 回 content_brief，不生成 draft |
| `hook_route` / `body_information_density_score` 缺失 | 回 draft，不进入视觉计划 |
| `visual_plan_status` 缺失 | 回 visual_plan，不进入质检 |
| `review_status` 缺失 | 回 quality_review，不进入平台包装 |
| `delivery_status` 缺失 | 回 platform_package / content_delivery_record，不生成最终 HTML |
| `image_assets_status` 缺失 | 最终 HTML 可生成但必须显示 unknown / pending，不得声称图片已齐 |
