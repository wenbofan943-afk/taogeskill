# 公开首页与发布文档复核复盘（2026-07-14）

> 归档状态：historical_audit_only；2026-07-18 移出当前解释热路径，不得作为 current 合同真源。

> 状态：已编译为发布门禁
> 适用范围：公开 README、安装、更新、状态、Release Notes、CHANGELOG、NOTICE 等入口文档。

## 结论

`README.md` 的早期基线同时承担公开 landing、历史产品路线、内部编排和未来设想。后续 Alpha 发版只更新了顶部版本与 Alpha 提示，结构 / 链接 / 隐私检查均可通过，却仍让普通 GitHub 访问者读到已废止的产品口径。

这不是单次漏改，而是发布链缺少“公开入口文档是否已按当前候选复核”的机器合同。

## 已采取的修订

1. 首页只保留当前定位、产物、快速开始、安装验证、边界、反馈和深入阅读入口。
2. 历史路线、内部编排和具体产品合同回到分区索引与 Git 历史，不再在首页重复陈述。
3. 新增 `docs/governance/public-entry-document-review.yaml`：逐份记录当前候选版本的复核状态、理由、必需词与已知废止词。
4. 新增 `tools/validate-public-entry-doc-review.ps1`：对当前树或公开候选包执行复核合同；内置“首页重新出现旧版本 / 旧产品词”负例。
5. 将该检查接入 `public_entry_document_gate`、公开 build profile、公开候选校验和发布必读清单。

## 未来规则

版本变更、公开定位变更、安装 / 更新流程变更、公开边界变更或首页结构性改动时，必须先更新这份复核合同；公开候选包未通过 `public_entry_document_gate`，不得进入发布。

历史事实不因首页精简而丢失：其可审计来源是 Git 历史；需要解释时，应新建 explanation 文档并从分区索引进入，而不是把历史全文重新堆回 README。
