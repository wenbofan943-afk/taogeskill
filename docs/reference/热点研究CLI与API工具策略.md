# 热点研究 CLI 与 API 工具策略

> 状态：传播研究方法论单元
> 主责：定义热点研究可以借助哪些现成 CLI / API，不自建完整工程，只支撑文案层的热点发现、摘要、评分、选题和草案生成。
> 边界：本文不引入自动发布，不接平台发布 API，不建设后台系统，不改变产品代码、服务器、数据库、积分、license 或发版链路。

---

## 一、核心判断

当前需求不是做一个新系统，而是给传播研究提供“外部信息输入”。

v1.9.1 阶段产品能力固定，传播动作只做到：

```text
热点发现
-> 热点评价
-> 账号母题匹配
-> 文案 / 脚本草案
-> dbskill 质检
-> 涛哥人工确认
-> 人工发布
```

因此工具策略是：

```text
能用现成 API / CLI 就不用自己搭工程。
能手动触发就不做定时任务。
能输出 Markdown 就不建数据库。
能人工确认就不做自动发布。
```

## 二、工具分层

### 1. 实时 AI 搜索层

用途：

1. 找最近热点。
2. 查某个话题背景。
3. 找对标案例。
4. 找行业讨论和用户痛点。
5. 给热点候选补事实来源。

候选工具：

| 工具 | 适合做什么 | 备注 |
|---|---|---|
| Tavily | AI agent 实时搜索、网页摘要、搜索结果内容提取 | 官方文档提供 Python SDK，适合后续做轻脚本 |
| Perplexity Search / Sonar | 实时搜索、带来源的研究和问答 | 适合热点研究和资料核验 |
| Exa | 语义搜索、相似内容、深度资料和对标案例 | 适合找“同类表达”和深度文章 |

官方资料：

- Tavily Docs: https://docs.tavily.com/welcome
- Perplexity API Docs: https://docs.perplexity.ai/docs/getting-started/overview
- Perplexity Search API: https://docs.perplexity.ai/docs/search/quickstart
- Exa Docs: https://exa.ai/docs/reference/getting-started

第一阶段建议：

```text
优先用 Codex 自带联网搜索 + 人工筛选。
如果需要稳定批量跑，再考虑 Tavily 或 Perplexity API。
```

---

### 2. 新闻和公开趋势层

用途：

1. 查新闻热点。
2. 看一个话题是否在持续发酵。
3. 找公共讨论的时间线。
4. 辅助判断热点是否值得蹭。

候选工具：

| 工具 | 适合做什么 | 备注 |
|---|---|---|
| GDELT | 开放新闻数据库、趋势观察、事件和知识图谱分析 | 免费开放，适合新闻趋势研究 |
| NewsAPI | 关键词新闻、top headlines、新闻源聚合 | 使用简单，适合轻量热点池 |
| Google Trends / SerpApi Trends | 搜索趋势、关键词热度变化 | pytrends 非官方，SerpApi 更稳但付费 |

官方资料：

- GDELT Data / Analysis Service: https://www.gdeltproject.org/data.html
- GDELT DOC 2.0 API: https://blog.gdeltproject.org/gdelt-doc-2-0-api-debuts/

第一阶段建议：

```text
先不用新闻 API。
需要查公共新闻热度时，优先手动搜索 + GDELT 辅助。
```

---

### 3. dbskill 质检层

用途：

dbskill 不负责找热点，不负责发布，只负责内容把关。

| 阶段 | 推荐 skill | 作用 |
|---|---|---|
| 热点共鸣判断 | `dbs-spread` | 判断热点背后的情绪和传播心理 |
| 选题形式判断 | `dbs-content` | 判断选题适合短视频、图文、文章还是组合 |
| 文稿共鸣 | `dbs-resonate` | 判断初稿有没有刺中核心 |
| 短视频开头 | `dbs-hook` | 检查前 5 秒话题、Hook 和可信度 |
| 标题方向 | `dbs-xhs-title` | 小红书标题和封面方向 |
| 对标分析 | `dbs-benchmark` | 拆同类内容和竞品表达方式 |

## 三、推荐执行方式

第一阶段不要搭工程，只做手动触发。

### 每次热点研究输入

```text
时间范围：今天 / 本周 / 最近 30 天
平台范围：抖音 / 小红书 / 视频号 / B站 / 微博 / GitHub / 新闻
目标人群：二手本地经营者 / 本地商家 / 直播运营 / 账号工作室 / 小团队老板
账号母题：从 5 个账号母题中选
输出数量：候选热点 10-20 个，正式选题 1-3 个
```

### 每次热点研究输出

```text
热点候选池
热点评分表
可做选题卡
不建议蹭的热点和原因
文案草案
dbskill 质检结论
人工确认项
```

## 四、轻量脚本边界

允许后续做轻量脚本，但必须满足：

1. 只读外部公开信息。
2. 只输出 Markdown / CSV / JSON 到 ``。
3. 不写数据库。
4. 不做定时任务。
5. 不调用平台发布 API。
6. 不自动登录任何平台。
7. 不触碰本项目客户端、服务器、积分、license、Cookie、真实客户数据。

可以做的轻量脚本：

```text
fetch_hot_topics.ps1
score_hot_topics.py
generate_topic_cards.py
```

这些脚本如果出现，默认是传播研究辅助工具，不是产品工程代码。

## 五、第一阶段工具选择

建议顺序：

1. **先用 Codex 联网搜索 + 本文评分规则**
   成本最低，适合验证流程。

2. **需要稳定重复跑时，引入 Tavily 或 Perplexity API**
   用来批量找热点、摘要网页和补来源。

3. **需要找相似表达和深度案例时，引入 Exa**
   用来找对标内容、同类工具表达和深度文章。

4. **需要公共新闻趋势时，引入 GDELT**
   用来判断热点是否只是局部噪声。

暂不建议：

1. 自建热点库。
2. 自建内容后台。
3. 自动发布。
4. 自动抓平台后台数据。
5. 自动维护多平台账号。
