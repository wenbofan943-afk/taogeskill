# R2 运行模型执行规范

> 状态：active  
> 所属路线：R2 运行模型与分支封锁  
> 主责：把 R2-C01 到 R2-C20 编译成运行时可执行规范，供 `propagation-router`、`final-delivery-builder` 和后续 sample / dry-run 使用。  
> 边界：本文件不实现并发执行器、不写 validator 脚本、不做自动发布、不接外部 API。

---

## 1. 适用场景

当用户出现以下情况时，必须按本规范处理：

```text
三篇都做 / 都要 / 这几个都跑。
跨账号多选。
产品开发任务中突然提出内容生产。
接着上次 / 活了吗 / 刚刚断了。
需要恢复 parent / child session。
需要重建账号索引或 all_runs。
```

---

## 2. 状态命名空间

R2 状态必须带命名空间，不得使用裸状态。

```text
run_*：单个 session / child_run 的运行状态。
branch_request_*：用户多分支请求的处理状态。
fan_out_*：parent 拆 child 的展开状态。
fan_in_*：parent 汇总 child 的收口状态。
run_lock：只描述运行锁。
blocked_reason：解释阻断原因，不发明新状态。
```

允许值：

```text
run_planned / run_active / run_waiting_human / run_completed / run_blocked / run_archived / run_discarded
branch_request_none / branch_request_requested / branch_request_confirmed / branch_request_deferred / branch_request_cancelled / branch_request_completed
fan_out_none / fan_out_planned / fan_out_running / fan_out_completed / fan_out_blocked
fan_in_none / fan_in_waiting_children / fan_in_ready / fan_in_completed / fan_in_blocked
run_lock_free / run_lock_acquired / run_lock_conflict / run_lock_released
```

禁止：

```text
planned / active / completed 这类裸状态。
branch_request_needs_dedupe 这类临时状态。
child_planned_blocked 这类混合状态。
```

---

## 3. Parent / Child

多选必须拆 child session：

```text
一个 topic_id -> 一个 content_run_id -> 一个 session 目录。
多个 topic_id -> 多个 child session。
parent session 只做分支规划、状态汇总和索引，不合并多篇正文。
```

父 session 必须记录：

```yaml
branch_request:
  branch_request_id:
  branch_mode: fan_out
  task_context_type:
  parent_session_id:
  source_artifact:
  requested_items:
    - topic_id:
      target_account:
      planned_child_session_id:
      child_run_status:
  branch_request_status:
  fan_out_status:
  fan_in_artifact:
parent_close_policy:
  on_parent_completed: keep_children
  on_parent_archived: ask_human / keep_children / archive_children
  on_parent_cancelled: ask_human / cancel_planned_children / keep_active_children
  on_parent_failed: keep_children_and_mark_parent_blocked
```

默认策略：

```text
父 completed：只表示分支规划完成，不表示 child 全部完成。
父 archived：必须问是否保留 child。
父 cancelled：未启动 child 可取消，已启动 child 要人工决定。
父 failed：不自动删除 child。
```

## 4. R2 操作合同

| 操作 | 必需输入 | 必产输出 | 状态变化 |
|---|---|---|---|
| `detect_branch_request` | 用户原话、`task_context_type`、current artifact | `human_decision_payload` 或 `branch_request` | `branch_request_requested` |
| `plan_fan_out` | parent manifest、topic_id 列表、账号确认状态 | parent manifest + branch ledger | `fan_out_planned` |
| `create_child_session` | branch_request、topic_card、账号档案、对象档案 | child session + child manifest | `run_planned` |
| `start_child_run` | child manifest、topic_card 完整性检查 | run_lock + checkpoint | `run_active` |
| `complete_child_run` | child final_delivery、quality_review、manifest | child state_transition + parent 计数 | `run_completed` |
| `block_child_run` | 失败阶段、blocked_reason、checkpoint | child manifest + parent blocked_count | `run_blocked` |
| `build_fan_in_summary` | 所有 child manifest | `intermediate/branch-summary.md` + indexes | `fan_in_completed` |
| `resume_run` | manifest、最新 checkpoint、state_transitions、run_lock | resume_report | `run_active / run_blocked` |
| `release_lock` | run_lock、state_transition | manifest 更新 | `run_lock_released` |

每个操作都必须写清：

```text
input_artifacts
output_artifacts
state_transition
failure_mode
resume_hint
```

## 5. 必产运行证据

R2 编译后，新样本必须具备：

```text
manifest.yaml
intermediate/checkpoints/
intermediate/branch-request-ledger.md（有分支时）
state_transitions（manifest 内或独立追溯块）
human_interrupt / human_decision_payload（有人类决策时）
intermediate/00-execution-trace.md
```

checkpoint 命名：

```text
CKPT-{session_id}-{stage}-{YYYYMMDD-HHMMSS}.md
```

checkpoint 必须在以下时机生成：

```text
主要 stage 完成后。
人类停顿点创建时。
断流恢复前。
error_before_retry 时。
```

---

## 6. 恢复读取顺序

用户说“接着上次 / 活了吗 / 刚刚断了”时，读取顺序固定：

```text
STATUS.md
-> 工作流状态记录.md
-> current_artifact
-> session manifest.yaml
-> 最新 checkpoint
-> state_transitions
-> branch-request-ledger.md
-> execution_trace
-> 实际 intermediate / deliverables 文件
```

恢复输出必须包含：

```yaml
resume_report:
  session_id:
  content_run_id:
  current_stage:
  last_completed_stage:
  latest_checkpoint:
  pending_human_interrupt:
  run_lock_status:
  recommended_resume_action:
  do_not_rerun:
  safe_to_rerun:
  blocked_reason:
```

禁止：

```text
只凭聊天记忆恢复。
重跑已完成 child。
跳过 run_lock 直接改正文。
把无 checkpoint 的恢复说成脚本级断点续跑。
```

## 7. 分支台账

路径：

```text
accounts/{account}/runs/{parent_session_id}/intermediate/branch-request-ledger.md
```

每条记录：

```yaml
ledger_entry:
  entry_id:
  branch_request_id:
  at:
  actor_type:
  action:
  input:
  output:
  from_status:
  to_status:
  affected_children:
  note:
```

必须登记：

```text
branch_request_requested
fan_out_planned
child_session_created
child_started
child_completed
child_blocked
child_archived
fan_in_waiting_children
fan_in_ready
fan_in_completed
parent_archived
resume_attempted
```

---

## 8. 编译后验收边界

R2 编译后只允许：

```text
static check
dry-run
sample run
```

不允许：

```text
完整真实测试。
真正并发执行。
自动 resume runner。
validator 脚本实现。
自动发布或平台登录。
```
