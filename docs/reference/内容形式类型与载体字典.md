# 内容形式类型与载体字典

> 状态：传播研究方法论单元
> 主责：统一内容 Brief 和选题卡里的内容形式、内容类型、外挂项和发布载体口径。
> 边界：本文服务 AI 判断，不要求用户填复杂表；不接平台发布 API，不自动发布，不改变产品功能。

---

## 一、为什么要分层

不要把“内容形式、内容类型、表现手法、发布渠道、素材来源”混在一起。

```text
内容形式：最终成品长什么样。
内容类型：这条内容用什么结构讲。
外挂项：发布前按平台追加的包装。
发布渠道：发到哪里，属于文案后的外挂治理。
素材来源：内容材料从哪里来。
```

当前阶段，Brief 只必须稳定输出：

```text
内容形式
内容类型
外挂项建议
```

发布渠道可以建议，但不是 Brief 的核心必填项；它主要影响后续标题、封面、发布描述、话题标签和平台适配。

第一阶段新增独立处理层：

```text
多平台分发包装：同一条视频主体不重做，只按平台生成封面标题、视频标题、发布描述和话题标签。
```

## 二、内容形式 Format

内容形式回答：

```text
这条内容最终长什么样？
```

当前允许值：

| 内容形式 | 说明 | 适合场景 |
|---|---|---|
| 短视频口播 | 真人出镜或纯口播脚本 | 观点、热点、信任、破圈 |
| 短视频录屏 | 以屏幕操作、产品页面或数据结果为主 | 产品演示、教程说明、功能展示 |
| 图文 / 轮播 | 多图、长图、卡片或小红书式轮播 | 方法、清单、步骤、案例复盘 |
| 长文 / 公众号 | 文章型内容 | 深度分析、理念说明、复盘 |
| 官网说明 / FAQ | 官网、下载页、GitHub 说明、申请页说明 | 对外口径、申请解释、产品边界 |

第一阶段制作口径：

```text
默认只制作“短视频口播”。
图文 / 轮播、长文 / 公众号、朋友圈、社群文案、官网说明 / FAQ 先保留为未来路由。
热点选题和 Brief 可以判断这些形式是否更适合，但第一阶段不展开对应制作办法。
```

不作为一级内容形式：

| 名称 | 正确归类 |
|---|---|
| 案例拆解 | 内容类型 |
| 产品演示 | 内容类型 |
| 短视频画中画 | 外挂项 / 表现手法 |
| 直播切片 | 素材来源或短视频口播/短视频录屏的一种生产方式 |
| VLOG | 当前不作为默认形式，除非账号明确走生活化记录 |

---

## 三、内容类型 Type

内容类型回答：

```text
这条内容用什么结构讲？
```

当前允许值：

| 内容类型 | 说明 |
|---|---|
| 观点评论 | 表达立场、判断和解释 |
| 热点解读 | 从热点切入，解释背后的行业或人群问题 |
| 案例拆解 | 用一个具体案例拆机制、问题和方法 |
| 教程说明 | 讲步骤、流程、做法 |
| 产品演示 | 展示工具如何完成一个任务 |
| 用户反馈复盘 | 基于用户反馈讲产品判断和改进 |
| FAQ 答疑 | 回答常见疑问、边界和申请问题 |
| 清单总结 | 给出列表、框架、避坑点 |

---

## 四、外挂项 Add-ons

外挂项回答：

```text
这条内容发布前还需要什么包装？
```

当前允许值：

| 外挂项 | 说明 | 常见适用 |
|---|---|---|
| 画中画 | 给短视频口播配 AI 图、截图、隐喻图或证据图 | 短视频口播 |
| 封面标题 | 封面上的短标题 | 短视频、图文、B站 |
| 内容标题 | 正文标题或视频标题 | 全部 |
| 发布描述 | 发布时配文 | 抖音、视频号、小红书、B站 |
| 话题标签 | 平台标签 | 抖音、小红书、视频号、B站 |
| 平台适配版本 | 同一内容按平台改短、改标题、改封面 | 多平台分发 |

外挂项由 AI 建议，不默认要求用户手填。

多平台分发包装默认由 `platform-packaging-adapter` 执行，位置在文案 + 视觉质检通过之后、涛哥最终人工确认之前。

## 五、发布渠道 Channel

发布渠道回答：

```text
发到哪里？
```

当前可选：

```text
抖音
视频号
小红书
公众号
B站
GitHub
官网 / 下载页
朋友圈
```

规则：

```text
发布渠道不是内容创作的第一决策。
发布渠道主要影响外挂项治理。
同一内容可以有多个平台适配版本。
本项目不自动发布、不接平台 API、不自动登录平台后台。
```

第一阶段默认平台：

```text
抖音
快手
小红书
视频号
```

包装差异：

| 平台 | 包装目标 |
|---|---|
| 抖音 | 停留、热点情绪、推荐流点击 |
| 快手 | 真实感、社区信任、垂类识别 |
| 小红书 | 搜索、收藏、方法感 |
| 视频号 | 信任、转发、熟人链路 |

## 六、素材来源 Source Material

素材来源回答：

```text
内容材料从哪里来？
```

当前常见来源：

```text
热点新闻 / 热榜
行业报告 / 平台公开资料
评论区公开互动
直播间公开互动
用户试用反馈
测试机反馈
产品截图 / 录屏
客户聊天摘要（必须脱敏）
```

素材来源不是内容形式。

---

## 七、AI 判断规则

默认由 AI 根据以下信息给出建议：

```text
账号档案
账号阶段
文案策略
目标人群
内容目标
产品露出比例
热点事实
选题复杂度
是否需要展示产品
```

只有出现以下情况才问涛哥：

```text
口播和图文都明显可行，但目标不同。
产品演示是否需要真人出镜不清楚。
官网说明 / FAQ 是否要对外发布不清楚。
平台适配成本过高，需要取舍。
```

默认问题要简单：

```text
这条更想做口播、图文，还是长文？
```

不要让用户填写“内容形式 / 内容类型 / 表现手法 / 渠道 / 素材来源”一整套复杂字段。
