# 账号档案完整性检查表

> 状态：传播研究规则
> 主责：定义热点创作 skill 启动前，账号档案必须放在哪里、怎么命名、哪些字段必须齐全、缺字段时怎么补。
> 边界：本文只约束传播研究和内容创作 skill，不改变客户端、服务器、数据库、积分、license、构建或发布链路。

---

## 一、固定位置

账号档案统一放在：

```text
accounts/
```

每个账号一个 Markdown 文件。

命名规则：

```text
accounts/{账号名}/account_profile.md
```

示例：

```text
accounts/示例行业观察号/account_profile.md
accounts/示例服务账号/account_profile.md
accounts/示例垂类经营号/account_profile.md
accounts/示例观点账号/account_profile.md
accounts/示例评论账号/account_profile.md
```

禁止把账号档案散落到根目录、临时报告、聊天记录、脚本目录或 skill 目录里。

## 二、热点创作前置门禁

用户呼喊“热点创作 / 热点 skill / 找热点 / 今天做什么内容”时，先执行：

```text
1. 询问或识别本轮账号。
2. 到 accounts/ 查找对应账号档案。
3. 如果账号档案不存在，先创建草案，并通过对话补齐用户可回答字段。
4. 如果账号档案存在，先检查 P0 字段是否齐全。
5. 如果本轮账号与上一轮账号不同，或用户明确要求换账号，先做账号档案对齐确认。
6. 用户可回答字段缺失时，不进入热点搜索；先问涛哥补齐。
7. AI 归纳字段缺失时，先由 AI 基于账号基础、同类账号和行业内容规律生成候选，再让涛哥确认。
8. P0 齐全、P1 有合理默认或明确待确认，并且本轮账号档案已对齐确认后，才能进入热点发现。
9. R5-H5：P0 对齐后仍须运行账号启动检查；只补当前任务需要的发布平台 / 时长、受众优先级、高风险口径和账号雷达策略，并生成 session 账号快照。
```

如果用户明确说“先用临时账号跑一轮”，可以输出临时结果，但必须标注：

```text
本轮使用临时账号假设，结果不得沉淀为正式账号策略。
```

### 换账号后的档案对齐确认

账号档案存在且 P0 齐全，不代表本轮可以直接开搜。换账号经营时，必须先把档案摘要给涛哥看一次。

这样做的原因：

```text
账号实际情况可能已经变化。
旧档案会影响热点选择、内容语气、目标人群、产品露出和内容禁区。
如果不先对齐，后面即使文案写得完整，也可能不是这个账号该发的内容。
```

对齐时不要让用户看完整字段表，只给关键摘要：

```text
账号定位
业务目标
目标人群
核心业务 / 服务
当前阶段
账号母题
内容禁区
转化路径
产品 / 服务露出比例
```

推荐引导语：

```text
我先把“{账号名}”的账号档案和你对齐一下。这样做是为了避免账号实际情况已经变了，但我还按旧画像找热点，最后选题、语气、产品露出或禁区不符合你的预期。

我读到的当前档案是：{账号定位}；主要讲给 {目标人群}；当前目标是 {业务目标}；长期母题是 {母题摘要}；内容里产品 / 服务露出应该 {露出比例}；明确不能碰的是 {禁区摘要}。

如果这版没变化，你直接回复“认可”或“同意”，我就继续做产品对象检查和热点研究；如果有变化，直接说要改哪里。
```

允许用户回复：

```text
认可
同意
没变化
就按这个
目标人群改成……
这条不要引导……
禁区加上……
```

用户回复认可后，记录：

```text
account_profile_confirmed_for_session = yes
account_profile_confirmed_at = YYYY-MM-DD
```

随后不要直接进热点搜索。先运行 `account_startup_check`：每轮最多 3 个口语问题；热点任务缺视觉身份时仅记录为非阻断项，视觉交付才补视觉身份。账号切换时必须新建当前账号的快照，不得复用上一个账号的策略、词库或候选。

用户指出变化时，先修订账号档案或在本轮 session 里标记临时覆盖，再进入热点研究。

### 交互补档规则

补账号档案时，不要直接问字段名，不要问“业务目标是什么 / 账号母题是什么 / 转化路径是什么”这种黑话。

账号档案分两步建：

```text
第一步：用户口语回答基础事实，AI 归纳落盘。
第二步：AI 基于基础事实、联网调研和同类账号规律生成策略字段候选，让涛哥确认。
```

用户大概率能讲清楚的字段，可以通过口语问答获取。用户大概率讲不清楚的字段，不要开放追问，而是由 AI 先调研和生成候选。

必须使用口语化引导语：

```text
一次最多问 3 个问题。
问题必须让普通人能直接回答。
用户回答后，先做落盘质检；质检通过后，再由 agent 翻译成结构化字段落盘。
不要把用户原话整段堆进档案；先归纳，再写入对应字段。
不确定的内容写入“待确认”，不要编造。
```

### 落盘质检规则

用户回答不能直接落盘。每条回答都必须先过质检：

```text
1. 是否回答了当前问题。
2. 是否能归入目标字段。
3. 是否和账号定位、目标人群、内容禁区冲突。
4. 是否是玩笑、情绪、个人偏好、临时想法或无关信息。
5. 是否足够具体，能支撑后续热点筛选。
```

质检结论分三类：

| 结论 | 处理 |
|---|---|
| 可落盘 | 归纳后写入对应字段 |
| 需追问 | 不落盘，换一种口语问法追问 |
| 不相关 / 无效 | 不写入正式字段，可放入待确认备注；必要时提醒用户这不是该字段答案 |

典型禁止：

```text
不能把“我喜欢小狗”写成业务目标。
不能把“所有人”写成目标人群。
不能把单个热点写成账号母题。
不能把临时想法写成长期账号定位。
不能把“做大做强”“搞流量”“随便看看”原样写入字段。
不能把灰产、自动骚扰、绕平台规则的表达写成可用业务能力。
```

遇到无关回答时，推荐追问：

```text
“这个回答更像个人偏好，不太像这个账号的目标。我换个问法：这个账号做内容，最后是想涨粉、接咨询、卖服务、建立专业信任，还是记录个人兴趣？”
```

### 落盘质检表

补档案时，可以先在回复里短暂展示质检结果，再落盘：

```markdown
## 落盘质检
| 目标字段 | 用户回答 | 质检结论 | 处理 |
|---|---|---|---|
| 业务目标 | 我喜欢小狗 | 不相关 | 不落盘，追问 |

## 本次拟落盘
- 业务目标：待确认
```

## 三、字段分级

### P0：缺失则不能开始热点创作

| 字段 | 说明 |
|---|---|
| 账号名 | 对外账号名称 |
| 账号定位 | 这个账号在用户心里扮演什么角色 |
| 业务目标 | 这个账号最终服务什么业务 |
| 目标人群 | 内容主要讲给谁 |
| 核心业务 / 产品 / 服务 | 账号承接什么服务或产品 |
| 当前阶段 | 起号 / 流量 / 信任 / 转化 / 复盘 |
| 账号母题 | AI 根据基础事实和调研生成的长期内容方向，需涛哥确认 |
| 内容禁区 | 不能蹭什么、不能怎么说 |
| 转化路径 | 用户看完后希望去哪里或做什么 |

### 字段来源分层

| 字段 | 来源方式 | 说明 |
|---|---|---|
| 账号名 | 用户直接回答 | 用户通常能答清楚 |
| 账号定位 | 用户口语回答，AI 归纳 | 可以问角色，不问抽象定位 |
| 业务目标 | 用户口语回答，AI 归纳 | 可以问想完成什么 |
| 目标人群 | 用户口语回答，AI 归纳 | 可以问讲给谁听 |
| 核心业务 / 产品 / 服务 | 用户口语回答，AI 归纳 | 可以问别人看完找你干嘛 |
| 当前阶段 | 用户选择，AI 辅助判断 | 起号 / 流量 / 信任 / 转化 / 复盘 |
| 转化路径 | 用户口语回答，AI 归纳 | 可以问看完下一步做什么 |
| 内容禁区 | 用户口语回答 + AI 风险补充 | 可以问哪些话不能说 |
| 账号母题 | AI 调研归纳，涛哥确认 | 不直接问用户“母题是什么” |
| 可蹭热点类型 | AI 调研归纳，涛哥确认 | 基于账号、人群、行业和平台规律 |
| 禁蹭热点类型 | AI 调研归纳，涛哥确认 | 基于账号风险和平台风险 |
| 产品露出比例 | AI 建议，涛哥确认 | 基于账号阶段和文案策略 |
| 成功指标 | AI 建议，涛哥确认 | 基于账号目标和内容策略 |
| 内容形式偏好 | AI 建议，涛哥确认 | 参考内容形式字典，不让用户填复杂表 |
| 画中画风格 | AI 建议，涛哥确认 | 只在短视频口播/画中画场景使用 |
| 常用表达 / 禁用表达 | AI 从历史内容和用户反馈归纳，涛哥确认 | 新账号可先待确认 |

### P0 字段问法

| 字段 | 内部含义 | 口语化问法 | 可接受答案形态 | 落盘规则 |
|---|---|---|---|---|
| 账号名 | 对外账号名称 | 这次给哪个账号做？比如“示例行业观察号”还是“示例垂类经营号”？ | 一个明确账号名 | 写入 `账号名`，并按 `accounts/{账号名}/account_profile.md` 建档 |
| 账号定位 | 账号在用户心里的角色 | 这个号主要扮演什么角色？是观察员、服务号、方法论号，还是个人观点号？ | 一句话角色描述 | 写成“账号定位”，不要写成长段宣言 |
| 业务目标 | 账号在矩阵里的任务 | 这个号主要想完成什么？涨粉、建立信任、接咨询、做转化，还是沉淀观点？ | 1-3 个目标 | 写成“业务目标”；注意不等于每条内容都卖产品 |
| 目标人群 | 内容主要讲给谁 | 这个号主要讲给谁听？本地经营者、做决策用户、AI 从业者、普通老板，还是别的人？ | 人群列表 | 写成“目标人群”，不要泛写“所有人” |
| 核心业务 / 产品 / 服务 | 账号能承接什么 | 如果有人看完内容想找你，你希望他找你干嘛？咨询什么、买什么、申请什么、了解什么？ | 服务 / 产品 / 申请 / 咨询项 | 写成“核心业务 / 产品 / 服务” |
| 当前阶段 | 账号当下运营阶段 | 这个号现在更像是起号、拉流量、建立信任、接咨询，还是复盘维护？ | 从阶段词里选，可多选 | 写成“当前阶段” |
| 账号母题 | 长期反复讲的主题 | 不直接问用户。先根据账号定位、目标人群、业务目标、服务内容和行业调研生成 3-5 个候选，再问“这几个长期栏目方向，哪些保留、删除、合并或改名？” | 3-5 个候选 | 由 AI 生成候选，涛哥确认后写入“账号母题” |
| 内容禁区 | 不能碰的话题 / 表达 | 这个号有哪些话不能说？哪些热点坚决不蹭？哪些表达会伤账号？ | 禁区列表 | 写入“内容禁区”，必要时同步“禁蹭热点类型” |
| 转化路径 | 用户看完后下一步 | 用户看完以后，你希望他下一步做什么？关注、私信、评论、去主页、申请试用，还是找你咨询？ | 明确动作 | 写入“转化路径” |

### P1：建议补齐，缺失时可先带假设跑

| 字段 | 说明 |
|---|---|
| 可蹭热点类型 | 哪些热点适合这个账号 |
| 禁蹭热点类型 | 哪些热点会伤账号画像 |
| 产品露出比例 | 产品/服务在内容中的占比 |
| 常用表达 | 账号常用话术、语气、口头禅 |
| 禁用表达 | 听起来不像涛哥、像广告、像灰产的说法 |
| 成功指标 | 播放、完播、收藏、咨询、申请、私信等 |
| 内容形式偏好 | 短视频口播、短视频录屏、图文、长文、官网说明 / FAQ 等 |
| 画中画风格 | 证据图、隐喻图、情绪图、门店场景等偏好 |

### P1 字段问法

| 字段 | 口语化问法 | 落盘规则 |
|---|---|---|
| 可蹭热点类型 | 哪类热点这个号比较适合蹭？行业、政策、明星八卦、AI 工具、本地经营者案例，还是别的？ | 写入“可蹭热点类型” |
| 禁蹭热点类型 | 哪类热点这个号最好别碰？ | 写入“禁蹭热点类型” |
| 产品露出比例 | 这个号内容里产品/服务露出要轻一点，还是可以重一点？ | 写成“低 / 中 / 高 + 说明” |
| 常用表达 | 这个号说话有什么固定味道？常用什么说法？ | 写入“常用表达” |
| 禁用表达 | 哪些话一说就不像你，或者像广告、像灰产？ | 写入“禁用表达” |
| 成功指标 | 你看这个号成不成，主要看什么？播放、完播、收藏、咨询、申请，还是私信？ | 写入“主指标 / 辅助指标” |
| 内容形式偏好 | AI 先根据账号阶段和内容目标推荐，不直接让用户填复杂表；必要时只问“这条更想做口播、图文，还是长文？” | 写入“内容形式偏好”或待确认 |
| 画中画风格 | 这个号配图更适合证据图、冲突图、隐喻图、门店场景，还是情绪图？ | 写入“画中画风格” |

### 分批提问建议

如果账号档案从零创建，推荐三轮问完，不要一次问十几个：

```text
第一轮：账号名 / 账号定位 / 目标人群
第二轮：账号主要想完成什么 / 能承接什么服务 / 用户看完下一步做什么
第三轮：内容禁区 / 哪些表达不能碰
第四轮：AI 调研后给出账号母题、热点边界、形式偏好、成功指标候选，让涛哥确认
```

P1 字段可以在第一次跑完 1-3 条内容后逐步补，不必阻断热点创作。

### P2：长期优化字段

| 字段 | 说明 |
|---|---|
| 对标账号 | 可研究但不能照抄 |
| 历史爆款 | 后续复盘后补 |
| 历史踩坑 | 后续复盘后补 |
| 发布频率 | 后续运营稳定后补 |
| 发布复盘规则 | 后续接入发布观察后补 |

## 六、原则

热点不是独立存在的。热点必须先过账号，再过母题，最后才看产品能力。

```text
账号
-> 人群
-> 业务目标
-> 母题
-> 热点
-> 推导链
-> 内容
```

没有账号档案，热点创作就是乱蹭。
