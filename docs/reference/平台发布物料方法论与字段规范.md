# 平台发布物料方法论与字段规范

> 状态：平台包装字段基准  
> 主责：规定封面标题、视频标题、发布描述、话题标签在最终交付物里的职责、字段和展示方式。  
> 边界：本文件不定义平台算法，不承诺流量结果，不触发自动发布。

---

## 一、问题定义

最终交付物不能把“封面标题”和“视频标题”混成一个泛化的“标题”。

```text
封面标题：服务滑动 / 列表 / 推荐流里的第一眼停留，通常短、粗、视觉化。
视频标题：服务平台标题区、搜索、转发、收藏和内容理解，通常比封面标题更完整。
发布描述：服务补充语境、互动引导、边界说明和转发理解。
话题标签：服务平台归类、搜索和垂类识别，不负责替代标题。
```

## 二、调研依据

本规范参考公开平台创作资料和创作工具说明：

| 平台 | 可借鉴结论 | 来源 |
|---|---|---|
| B站 | 官方 UP 主创作手册把封面和标题分开讲；封面要求主体明确、文字少而大，标题要求精炼简洁、突出关键。 | `https://activity.hdslb.com/blackboard/static/pdf/20200331.pdf` |
| 抖音 | 创作服务平台支持发布管理、修改描述和封面，封面与描述是不同操作对象。 | `https://creator.douyin.com/upload` |
| 小红书 | 小红书内容的封面、标题、正文、标签共同影响点击与搜索；封面承担吸引点击和总结内容任务。 | 小红书公开创作 / 运营资料 |
| 视频号 | 视频号助手发布流程中封面、描述、话题属于不同发布项；封面用于吸引点击，描述补充观看理由。 | `https://channels.weixin.qq.com` 及公开操作说明 |

这些资料的共同点是：平台入口不是单字段，不能只交付一个“标题”。

## 三、字段职责

### `cover_title`

封面图上的短文字。

```text
短。
能一眼看懂冲突或收益。
适合放在封面图上，不遮挡主体。
不要承担完整解释。
```

### `video_title`

平台标题区使用的标题。

```text
比封面标题更完整。
承接口播 Hook。
保留关键词，利于搜索和转发理解。
不和封面标题完全重复，除非平台强依赖短标题。
```

### `publish_description`

发布页配文 / 简介 / 正文摘要。

```text
补上下文。
解释为什么值得看。
可加入互动问题。
可承载风险边界，不做事实夸大。
```

### `hashtags`

平台话题标签。

```text
少而准。
优先垂类、场景、主题。
不堆砌无关热门词。
不替代正文和标题。
```

## 四、平台差异

| 平台 | 封面标题 | 视频标题 | 发布描述 | 话题标签 |
|---|---|---|---|---|
| 抖音 | 强冲突、短句、停滑动 | Hook 延展，适合推荐流点击 | 一句话观点或互动问题 | 2-5 个，热点 + 垂类 |
| 快手 | 口语、真实、像人说话 | 少包装感，强调经验 | 说明给谁看、为什么这么说 | 身份、行业、场景 |
| 小红书 | 关键词 + 痛点 + 收藏理由 | 搜索友好，像笔记标题 | 可结构化，强调价值和适合谁 | 关键词、人群、场景 |
| 视频号 | 稳、可信、适合转发 | 观点完整，适合熟人链路理解 | 可稍长，补充观察和边界 | 少量主题标签 |
| B站 | 主题明确、少字大字 | 突出关键和亮点 | 简介可解释结构和看点 | 分区 / 标签服务检索 |

## 五、标准输出字段

每个平台必须输出：

```text
platform
cover_title_options
recommended_cover_title
video_title_options
recommended_video_title
publish_description_options
recommended_publish_description
hashtag_sets
recommended_hashtags
platform_notes
manual_publish_notes
```

禁止只输出：

```text
标题
内容标题
发布标题
```

如果为了人类阅读需要展示“标题”，也必须同时标明它到底是：

```text
封面标题 / 视频标题 / 笔记标题 / B站标题
```

## 六、最终 HTML 展示要求

最终 HTML 的平台物料区，每个平台至少展示：

```text
封面标题
视频标题
发布描述
话题标签
发布备注
```

如果只展示“标题”，判定为交付缺陷：

```text
platform_package_display_status = fail
revision_path = back_to_platform_package
```

## 七、质检清单

平台包装完成前必须检查：

```text
封面标题和视频标题是否分开。
封面标题是否适合放在画面上。
视频标题是否承接口播 Hook。
发布描述是否补足语境，而不是重复标题。
话题标签是否少而准。
各平台是否按平台语气调整。
最终 HTML 是否没有使用泛化“标题”替代二者。
```
