# AI 热点发现与关联评估方法论

> 状态：传播研究方法论单元
> 主责：定义 AI 如何发现热点、筛选热点、评价热点与本产品账号母题的关联度，并产出可落地内容任务。
> 边界：本文只指导传播研究和内容规划，不作为产品开发验收包，不作为工程实现计划，不直接修改客户端、服务器、数据库、积分、license 或发版链路。

---

## 一、核心目标

AI 找热点不是为了追流量，而是为了找到：

```text
目标人群正在关注的问题
-> 和账号母题有真实关系
-> 能自然引出公开互动分析工具的产品能力
-> 风险可控
-> 能转化成可发布内容
```

禁止把热点当作万能入口。热点只是放大器，不是内容本身。

本方法论只做到文案层：

```text
找热点
-> 评估关联度
-> 匹配账号母题
-> 生成选题
-> 选题确认后编译内容 Brief
-> 生成文案草案
-> dbskill 质检和微调
-> 涛哥人工确认
```

不做自动发布、不接平台发布 API、不自动登录平台后台、不自动评论 / 私信 / 互动。

默认不从热点直接跳到文案。标准流程分三段：

```text
阶段 1：热点探索
-> 阶段 2：选题 + 策略 + 推导链
-> 涛哥确认选题
-> 阶段 3：内容 Brief 编译 + IP 资产调用 + 文案创作 + dbskill 质检
-> 结束
```

除非涛哥明确说“不用中间确认，直接到文案草案”，否则在“选题 + 策略 + 推导链”层停下。

## 二、输入和输出

### 输入

每轮热点研究至少需要：

1. 当前固定产品能力。
2. 当前账号母题。
3. 目标人群。
4. 平台范围：抖音、视频号、小红书、B站、微博、公众号、知乎等。
5. 时间范围：今日、本周、近 30 天。
6. 禁区：灾难、违法犯罪细节、个人隐私、敏感身份识别、灰产导向、骚扰营销等。

### 热点时效输入

`time_range` 只是搜索范围，不等于热点判断。每轮还必须判断：

```text
hotspot_time_window：候选事实距离当前时间有多近。
hotspot_freshness_status：它是否还能被叫作热点。
content_position：这条内容最终应被定位成即时热点、行业趋势、常青问题、案例复盘还是方法论内容。
```

热点时效由三层共同决定：

```text
账号默认时效 -> 内容策略时效 -> 来源发布时间校验
```

账号决定默认搜索窗口，内容策略决定最终时效要求，来源发布时间负责兜底校验。

## 二点五、热点时效路由

热点不是“只要有人讨论就叫热点”。本项目按时间窗分层：

| hotspot_time_window | 时间窗 | 定义 | 默认用途 |
|---|---:|---|---|
| `breaking_0_24h` | 0-24 小时 | 刚发生，还在快速扩散 | 大热点破圈、即时评论 |
| `live_hot_1_3d` | 1-3 天 | 正在发酵，讨论活跃 | 最适合短视频热点切口 |
| `current_hot_3_7d` | 3-7 天 | 热度仍在，进入解释期 | 观点、解释、垂类转译 |
| `warm_trend_7_30d` | 7-30 天 | 不算即时热点，但仍有行业讨论 | 信任型、教育型、行业趋势 |
| `background_trend_30_90d` | 30-90 天 | 背景趋势，不应包装成热点 | 趋势背景、案例复盘 |
| `evergreen_90d_plus` | 90 天以上 | 常青问题或历史案例 | 方法论、复盘素材，不能叫热点 |

`hotspot_freshness_status` 判断：

```text
fresh_enough：符合当前账号和内容策略，可以叫热点。
aging_but_usable：时效偏旧，但可作为趋势或解释内容。
too_old_for_hotspot：不能叫热点，只能改成复盘或背景。
evergreen_only：只能作为常青问题或方法论素材。
unknown_time_blocked：来源时间不清，不能进入正式选题。
```

`content_position` 判断：

```text
breaking_hotspot：即时热点。
current_hotspot：当下热点。
industry_trend：行业趋势。
evergreen_problem：常青问题。
case_review：案例复盘。
methodology_content：方法论内容。
```

### 账号默认时效

| 账号类型 | 默认时间窗 | 说明 |
|---|---|---|
| 流量观点号 | `breaking_0_24h` 到 `current_hot_3_7d` | 例如示例行业观察号，越新越好 |
| B 端信任 / 转化号 | `current_hot_3_7d` 到 `warm_trend_7_30d` | 例如示例垂类经营号，可接受行业趋势 |
| C 端服务转化号 | `current_hot_3_7d` 到 `warm_trend_7_30d` | 例如示例服务账号，重点是场景和信任 |
| AI / 工具认知号 | `live_hot_1_3d` 到 `warm_trend_7_30d` | 例如示例观点账号，工具变化要新 |
| 低频垂类号 | `warm_trend_7_30d` 到 `background_trend_30_90d` | 例如示例评论账号，可以偏解释和复盘 |

### 内容策略时效

| 文案策略 | 允许时间窗 | 不能怎么做 |
|---|---|---|
| 大热点破圈型 | `breaking_0_24h` / `live_hot_1_3d` / `current_hot_3_7d` | 超过 7 天没有新进展，不再叫大热点 |
| 垂类获客型 | `current_hot_3_7d` / `warm_trend_7_30d` | 超过 30 天只能叫行业趋势或复盘 |
| 教育信任型 | `warm_trend_7_30d` / `background_trend_30_90d` / `evergreen_90d_plus` | 不强行写“今天刚发生” |
| 产品演示型 | `warm_trend_7_30d` / `evergreen_90d_plus` | 不能用旧热点制造紧迫感 |
| 复盘共创型 | `background_trend_30_90d` / `evergreen_90d_plus` | 必须明说是复盘，不叫热点 |

### 强制降级规则

1. 来源发布时间不清：`hotspot_freshness_status = unknown_time_blocked`，不能进入正式选题。
2. 超过 7 天的大众热点：不能叫“大热点破圈”，除非有新进展。
3. 超过 30 天的行业事件：不能叫“当下热点”，只能叫行业趋势、复盘或常青问题。
4. 超过 90 天的内容：默认 `evergreen_only`，只能做方法论、案例复盘或长期问题。
5. 如果内容定位被降级，选题卡必须同步改 `content_position` 和开头话术。

v1.9.1 阶段当前固定产品能力：

```text
公开互动分析工具
-> 评论区 / 直播间公开互动整理
-> AI 分析
-> 分析结果 / 重点互动 / 互动资产库
-> 免费学习版申请制
```

当前账号母题见：

- [docs/reference/账号母题与传播工作流.md](./账号母题与传播工作流.md)

### 输出

每轮热点研究输出：

1. 热点候选池。
2. 热点评分表。
3. 可做内容清单。
4. 不建议蹭的热点及原因。
5. 每条可做内容对应的账号母题、目标人群、产品能力和 dbskill 质检工具。

产物分层：

```text
来源池：长期方法论资产，不随单轮热点变化。
热点候选池：本轮搜索缓存，允许记录大量未通过热点。
热点评分表：筛选记录，判断桥接链、分数和去向。
自媒体选题库：长期内容资产，只放通过评分和策略判断的选题。
```

默认工作方式：

```text
搜回来的热点先写候选池。
评分结果写评分表。
通过 Topic Gate 的选题写选题库，`topic_status = topic_pending_human_choice`。
没有确认前不进入完整文案创作。
```

Topic Gate 不做纯静默过滤。未通过的热点不进入主推荐区，但必须保留在候选池或评分表里，写明过滤原因，便于复盘 AI 是否误杀。

## 三、热点来源

热点来源详表见：

- [docs/reference/热点搜索来源池.md](./热点搜索来源池.md)

AI 不直接随机搜索。每轮先按账号档案和来源池生成来源计划，再找热点。

简表：

| 来源 | 适合发现什么 | 对应池 | 注意 |
|---|---|---|
| 抖音热榜 / 视频号趋势 / 小红书热搜 / 微博热搜 | 大众热点、平台话题、情绪热点 | S / B | 要过滤泛娱乐、争议和低关联热点 |
| 行业媒体 / 垂直公众号 / B站知识区 | 行业趋势、工具趋势、AI 趋势 | B | 更适合做解释型内容 |
| GitHub Trending / Product Hunt / Hacker News | 工具、开源、AI 产品趋势 | B / S | 适合讲工具、AI、开源、免费版 |
| 微信群 / 客户反馈 / 测试机反馈 | 真实用户问题 | A/C | 转化价值高，但要脱敏 |
| 竞品账号 / 同类工具账号 | 表达方式、选题结构、用户评论 | A/C / B | 不照抄，只拆结构 |
| 我们自己的评论区 / 申请表 / 用户问答 | 一手需求和异议 | A/C | 适合做 FAQ、澄清和反误解内容 |

来源计划输出格式：

```markdown
## 本轮来源计划
| 池 | 本轮来源 | 为什么选 | 预计找什么 | 风险 |
|---|---|---|---|---|
```

## 三点五、三池热点雷达

账号档案不能直接等同于热点搜索关键词。

账号档案定义的是：

```text
这个账号能接住什么热点
什么热点会伤账号
什么热点能转成内容
```

因此热点发现必须同时运行三类雷达：

```text
S 池：公共大热点 / 大时代情绪 / 全民关注事件
B 池：行业热点 / 圈层热点 / 平台和市场变化
A/C 池：目标人群现场 / 账号母题附近的真实问题
```

### S 池：公共大热点

S 池不是从账号关键词里长出来的，而是从外部公共注意力池里发现。

典型方向：

```text
公共政策
社会情绪
消费变化
就业 / 创业焦虑
AI / 科技大事件
平台级变化
全民讨论话题
大品牌 / 大企业事件
```

S 池的价值是停人和破圈，但必须能往下桥接。

### B 池：行业 / 圈层热点

B 池来自账号行业和目标圈层，例如：

```text
行业 / 本地服务 / 本地经营者
AI 工具 / SaaS / 开源
短视频平台 / 直播 / 本地商家
账号矩阵 / 内容生产 / 用户反馈
```

### A/C 池：目标人群现场和账号母题

A/C 池来自更近的现场：

```text
用户评论
申请表
客户反馈
测试机反馈
同行评论区
账号母题里的长期问题
```

这类热点未必大，但转化和信任价值更高。

### 三池关系

```text
S 池负责大注意力。
B 池负责行业相关性。
A/C 池负责目标人群和账号母题。
```

不能只搜 B/A/C，否则容易错过 S 级公共热点；也不能只追 S，否则容易硬蹭。

## 四、热点初筛规则

AI 找到热点后，先做三层初筛。

### 1. 相关性初筛

问：

```text
这个热点能不能引出以下任一主题？
评论区管理
直播复盘
账号矩阵
用户反馈
AI 分析
公开互动
免费工具
小团队运营效率
```

如果完全不能，直接淘汰。

### 2. 风险初筛

以下热点默认不碰：

1. 灾难、伤亡、公共安全事故。
2. 违法犯罪细节。
3. 个人隐私、家庭纠纷、未成年人。
4. 容易引向人肉、开盒、身份识别的话题。
5. 容易把产品解释成批量骚扰、灰产引流、自动私信的话题。
6. 无法确认事实、容易造谣的话题。

### 3. 产品真实性初筛

问：

```text
如果观众看完这条内容下载软件，当前 v1.9.1 能不能支撑这个承诺？
```

如果不能，不能做。

可以讲“后续想法”，但必须明确不是当前能力。

## 五、热点关联度评分表

每个热点按 6 个维度评分，每项 0-2 分，总分 12 分。

| 维度 | 0 分 | 1 分 | 2 分 |
|---|---|---|---|
| 人群相关 | 和目标用户无关 | 间接相关 | 直接相关 |
| 母题相关 | 不属于任何账号母题 | 可勉强接入一个母题 | 天然属于一个母题 |
| 产品相关 | 产品接不上 | 只能间接提到产品 | 能自然展示产品能力 |
| 情绪强度 | 无明显情绪 | 有讨论但不强 | 有明显痛点、焦虑、反差或共鸣 |
| 风险可控 | 高风险，不建议做 | 有风险，需要克制表达 | 风险低，可正常做 |
| 可落地性 | 只能聊观点 | 能做一篇内容 | 能做系列内容或组合内容 |

评分结论：

```text
0-4 分：淘汰
5-7 分：只做轻观点或评论
8-10 分：可以进入正式选题
11-12 分：重点热点，可做图文 + 短视频 + 官网/下载页联动
```

## 五点五、大热点观点切口链

如果热点足够大，可以使用更大的公共注意力做入口，但必须有推导链，不能硬蹭。

三种内容层级：

```text
普通内容：A 目标人群现场 -> C 账号母题 -> D 产品能力
热点内容：B 大圈热点 -> A 目标人群现场 -> C 账号母题 -> D 产品能力
大传播内容：S 大公共热点 -> B 大圈热点 -> A 目标人群现场 -> C 账号母题 -> D 产品能力
```

五层含义：

| 层级 | 作用 | 示例 |
|---|---|---|
| S 大公共热点 / 时代情绪 | 抓最大注意力 | 阅兵式、强国、AI 战略、产业升级 |
| B 大圈热点 / 行业命题 | 承接公共讨论 | AI 进入小企业日常工作流 |
| A 目标人群现场 | 拉回具体人群 | 小团队只用 AI 写文案，却没整理真实反馈 |
| C 账号母题 | 进入长期表达 | AI 不只是生成内容，也能整理现场 |
| D 产品能力 | 落到工具 | 评论区 / 直播间公开互动分析 |

每一跳都要有依据：

```text
S -> B 是否成立？
B -> A 是否成立？
A -> C 是否成立？
C -> D 是否成立？
```

如果中间某一跳只能靠想象，不能进入文案。

不同来源池的桥接要求：

```text
S 池热点：必须写 S -> B -> A -> C -> D
B 池热点：必须写 B -> A -> C -> D
A/C 池热点：必须写 A -> C -> D
常青内容：必须写 A -> C -> D，并说明为什么无需热点
```

判断规则：

```text
能完整桥接：进入评分。
只能到 B，接不到 A/C：只做轻观点，不进正式选题。
能到 A/C，但 D 很硬：可以做账号观点，不强插产品。
任一关键跳只能靠想象：标记“不建议做”。
```

桥接质量比“是否写出桥接链”更重要。AI 很容易把硬蹭说成自然关联，所以必须给出桥接状态：

```text
未桥接：没有写出链路，不能进入选题。
弱桥接：有链路，但关键跳转依据薄，只能待补桥或降级轻观点。
可桥接：链路基本成立，可以进入评分。
强桥接：每一跳都有清楚依据，优先推荐。
硬蹭嫌疑：看似有链路，但主要靠口号、类比或情绪硬拉，淘汰或重写角度。
```

桥接质检必须说明：

```text
哪一跳最虚
这一跳的证据是什么
是否靠类比 / 口号 / 情绪硬拉
能否补桥
是否需要降级
```

推导链输出模板：

```text
热点来源池：
S 大公共热点：
B 大圈热点 / 行业命题：
A 目标人群现场：
C 账号母题：
D 产品能力：

每一跳依据：
哪一跳最虚：
如何补桥：
为什么不是硬蹭：
是否值得继续：
```

### 开头类型选择

大热点入口不一定需要反常识。

| 类型 | 适用 | 结构 |
|---|---|---|
| 反常识 Hook | 普通母题、窄切口、非热点内容 | 目标人群常识 -> 反差观点 -> 母题 |
| 大热点立场 Hook | 大公共热点、时代议题、强情绪热点 | 大热点事实 -> 我的立场 / 观点切口 -> 目标人群现场 |

原则：

```text
大热点靠公共注意力停人。
反常识靠认知反差停人。
两者不是必须叠加。
```

如果使用 S 层大热点入口，优先使用“大热点立场 Hook”，不要强行制造反常识。

## 七、AI 热点工作流

### Step 1：找热点

AI 每轮先从 S / B / A-C 三池收集 10-20 个候选热点，不做创作，只做候选池。

推荐比例：

```text
S 池：3-6 个
B 池：4-8 个
A/C 池：4-8 个
```

比例可以根据账号阶段调整：

```text
流量账号：S 池权重更高
转化账号：A/C 池权重更高
信任账号：B 池和 A/C 池权重更高
认知账号：S 池和 B 池权重更高
```

数量上限：

```text
候选热点：10-20 个，不多于 20 个
初筛保留：5-8 个，不多于 8 个
正式选题：3-5 个，不多于 5 个
```

停止条件：

```text
已收集 10-20 个候选热点
或已找到 5-8 个可进入评分的热点
或继续搜索 10 分钟仍没有更高质量热点
```

禁止无限搜索。热点质量不足时，说明“不足”，不要硬凑。

候选字段：

```text
热点名称
热点来源池
来源
时间
一句话事实
热度信号
事实核验等级
热点生命周期
可能关联母题
初步风险
```

事实核验等级：

```text
A：官方 / 权威来源确认
B：多来源交叉确认
C：单来源，需谨慎
D：传闻或不可核验，不建议做
```

热点生命周期：

```text
正在上升
高位扩散
开始衰退
已过时
适合复盘
```

### Step 2：初筛

先按三池来源标注，再按“三层初筛”淘汰明显不相关、风险过高、产品无法支撑的热点。

S 池热点不能因为“当前看起来不直接相关”立刻淘汰，必须先尝试 S -> B -> A -> C -> D 桥接。桥接失败再淘汰。

### Step 3：评分

对剩余热点做 12 分评分。

### Step 4：Topic Gate 选题前质检

评分后不要直接给涛哥选，先做 Topic Gate。

Topic Gate 检查：

```text
事实可靠
热点价值
账号匹配
桥接质量
受众情绪
产品 / 业务承接
风险可控
内容可执行
去重价值
```

展示策略：

```text
通过，待选择：进入主推荐区，给涛哥选。
降级轻观点：可折叠展示或只留评分表。
待补桥：不进主推荐区，记录最虚一跳和补桥方式。
淘汰：不展示给涛哥选择，只留原因。
归档复盘：不展示给涛哥选择，保留未来复盘价值。
```

原则：

```text
主决策区只给高质量选题。
被过滤内容不丢，必须留痕。
不要让涛哥在 20 条原始热点里做选择。
```

### Step 5：选题 + 策略 + 推导链

只对 8 分以上热点写选题卡，并判断文案策略：

```text
大热点破圈型
垂类获客型
教育信任型
产品演示型
复盘共创型
```

5-7 分热点只保留一句轻观点，不进入正式创作。

选题卡和推导链输出后必须停下，等待涛哥选择今天要做的选题。

### Step 6：内容 Brief 编译 + IP 资产调用

只有涛哥确认选题后，才进入文案创作。

人工选题结束的触发语包括：

```text
选 T001
选 T002
T003 就做这个
```

选中后，`topic_card` 状态改为“已选择，待生成 Brief”。只有字段完整的 topic_card 才能交给 `content-brief-compiler`。

正式写文案前，先用 `content-brief-compiler` 自动生成内容 Brief。Brief 是写作输入包，不是默认人工审批节点；只有缺信息或高风险时才停下来找涛哥确认。

```text
目标账号
目标人群
内容目标
文案策略
核心观点
事实核验等级
热点生命周期
账号母题
对应产品能力
必须调用的 IP 资产
不能承诺什么
不能怎么讲
推荐平台 / 形式
成功判断
人工确认点
```

content_brief 状态：

```text
brief_pass
brief_needs_human_confirm
brief_risk_high
brief_needs_format_change
brief_blocked
```

只有 `brief_status = brief_pass` 才能进入 Step 7。

如果可读取，优先调用认知推演工作流和短视频创作项目中的 IP 层资产：

```text
认知档案：涛哥怎么看问题
表达工坊：涛哥怎么说话
知识原子：涛哥有哪些观点、案例、判断
修改偏好：涛哥不喜欢哪些 AI 味
```

如果本轮未调用 IP 资产，必须说明原因。热点文案能力未来进入短视频创作 SaaS 时，应由 IP 层统一提供表达工坊、知识原子和认知档案。

### Step 7：文案创作 + dbskill 质检

给涛哥看到的文案必须已经完成 dbskill 质检，不给未质检初稿。

按内容阶段使用：

1. `dbs-spread`：判断热点背后的情绪和传播机制。
2. `dbs-content`：判断选题适合什么形式。
3. `dbs-resonate`：检查文稿是否有共鸣。
4. `dbs-hook`：检查短视频前 5 秒。
5. `dbs-xhs-title`：检查小红书标题和封面方向。

质检结果必须结构化检查：

```text
Hook 是否成立
观点是否像涛哥
产品是否硬插
事实是否可靠
是否有灰产误解
是否有 AI 味
是否值得发布
```

### Step 8：结束边界

标准任务结束于：

```text
已完成被选中选题的文案草案
已完成 dbskill 质检
已输出 `next_skill = human_confirm` 的人工确认项
```

本阶段不默认做发布复盘。

## 八、热点风险表达规则

可以说：

```text
这个事件说明，商家不能只看成交数据，也要看用户公开反馈。
这个现象提醒我们，评论区其实是最便宜的用户调研。
直播复盘不能只看在线人数，互动内容也要整理。
AI 不只是写文案，也能帮你整理已经发生的公开互动。
```

不要说：

```text
用这个工具去截别人客户。
用这个工具监控竞品账号。
批量抓取全平台用户。
自动私信这些人。
通过评论找到客户联系方式。
跨平台识别同一个人。
```

## 九、第一阶段执行标准

第一阶段不追求热点数量，追求判断质量。

每周做一次热点研究：

1. 候选热点：10-20 个。
2. 通过初筛：3-8 个。
3. 进入正式选题：1-3 个。
4. 实际发布：1-2 条。
5. 每条发布后做复盘。

第一阶段成功标准：

```text
不是播放量最高
而是能带来真实用户提问、申请、下载、试用和反馈
```
