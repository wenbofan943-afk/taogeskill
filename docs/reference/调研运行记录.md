# 调研运行记录

> 状态：传播工作流过程资产
> 主责：记录每轮热点调研的来源计划、实际搜索、来源质量、事实核验和进入候选池的原因。
> 边界：本文件不是新闻库，不长期保存所有原始链接；只保存能解释本轮选题来源和筛选依据的过程记录。

---

## 使用规则

1. 每轮热点调研必须先形成 `research_run_record`。
2. 记录“搜了哪里、为什么搜、搜到什么、为什么可信或不可信”。
3. 不把未经核验的传闻包装成事实。
4. 热点进入候选池前，必须能追溯到本文件中的来源计划或实际来源。
5. 涉及客户反馈、申请表、测试机反馈等真实材料时，只写脱敏摘要，不写姓名、电话、微信、账号私密信息。
