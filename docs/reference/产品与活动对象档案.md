# 产品与活动对象档案

> 状态：工作流对象边界基准
> 主责：在每轮内容创作前，定义这条内容服务的产品、服务、活动、观点对象或转化目标。
> 边界：本文件不是产品规格书，不替代对应产品项目的 README、AGENTS、ADR 或产品文档。

---

## 一、为什么需要它

账号档案回答“谁来说”：

```text
账号人设
受众
母题
语气
禁区
转化路径
```

产品/活动对象回答“这次说什么”：

```text
产品或服务是什么
本轮活动目标是什么
能承诺什么
不能承诺什么
希望用户下一步做什么
哪些证据可以用
哪些表达会越界
```

如果只有账号档案，没有产品/活动对象，AI 很容易把热点、账号观点和产品露出混在一起，最后出现硬蹭、强卖、承诺越界或交付口径不一致。

---

## 二、对象类型

### product_profile

用于长期存在的产品、服务或业务对象。

```text
公开互动分析工具
示例服务账号服务
本地经营者自媒体方法论
短视频创作 SaaS
认知推演工作流
```

### campaign_profile

用于一次阶段性推广、申请、活动或内容主题包。

```text
免费学习版申请
GitHub 下载页引流
某一轮本地经营者试用招募
某个热点专题周
某次直播预热
```

没有独立活动时，`campaign_profile_status = campaign_profile_not_used`。

## 四、第一版默认样例

### product_profile_id: P-public-interaction-tool

```text
product_name：公开互动分析工具
product_type：免费学习版工具 / 公开互动内容整理与 AI 分析工具
source_project：D:\OpenClaw\workspace\公开互动分析工具项目
positioning：帮助用户整理自己关注范围内的公开评论区和直播间互动内容，做学习、复盘、观察和内容管理。
target_audience：本地经营者、运营人员、自媒体账号、小团队经营者、需要复盘公开互动内容的人。
core_capabilities：公开互动整理、短视频评论分析、直播间互动分析、AI 分类、结果导出、互动资产沉淀。
allowed_claims：可以说整理公开互动内容、辅助复盘、帮助发现高价值评论、降低人工筛选成本。
forbidden_claims：不能说截流、灰产、批量私信、绕过平台限制、自动成交、获取手机号微信号、跨平台识别自然人。
conversion_goal：引导用户了解免费学习版、提交申请或联系涛哥了解适用边界。
cta_boundary：只引导申请、了解、试用和人工沟通；不承诺直接获客结果。
proof_assets：产品截图、导出表格示意、功能流程图、学习版申请说明。
risk_notes：必须强调公开内容、学习研究、账号自用或团队内部管理场景，避免被误解为骚扰或灰产工具。
product_profile_status：product_profile_ready
next_skill：hotspot-topic-research
```

### campaign_profile_id: C-free-learning-trial

```text
campaign_name：免费学习版申请
campaign_goal：让合适用户理解工具边界并提交申请。
campaign_period：长期有效，具体周期按产品项目状态调整。
target_audience：想做公开互动内容复盘、评论区管理、直播间互动整理或账号矩阵运营的小团队。
offer_or_hook：免费学习版、申请制、人工沟通适用场景。
allowed_claims：可以说免费学习版、申请审核、AI 分析额度、适合学习研究和内部复盘。
forbidden_claims：不能说永久免费无限用、无需审核、商用无后果、自动获客、自动触达。
conversion_path：内容触达 -> 了解官网/下载页 -> 提交申请 -> 人工沟通 -> 发放试用资格。
success_metric：申请量、有效沟通量、试用反馈质量。
risk_notes：申请制是为了控制成本、确认用途和降低滥用风险，不是制造门槛感。
campaign_profile_status：campaign_profile_ready
next_skill：hotspot-topic-research
```

## 五、使用规则

1. 每轮热点研究前，必须至少有一个 `product_profile`。
2. 如果本轮有明确推广活动，再绑定一个 `campaign_profile`。
3. 产品对象不清时，不进入热点研究；先补产品对象边界。
4. 传播工作流可以提出产品表达建议，但不能在本项目静默修改对应产品的正式规则。
5. 如果某次讨论形成正式产品决策，必须回写到对应产品项目。
