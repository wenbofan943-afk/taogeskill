# R3 图片资产执行规范

> 状态：active  
> 所属路线：R3 画中画与图片资产模型  
> 主责：把 R3-C01 到 R3-C90 编译成运行时可执行规范，供 `talking-head-image-pip`、`platform-packaging-adapter`、`cover-design-compiler`、`copywriting-quality-review`、`final-delivery-builder` 和后续 dry-run 使用。
> 边界：本文件不实现外部图片 API、不调用 Seedream、不建设图片服务、不自动发布。

## 1. 适用场景

当工作流进入以下场景时，必须按本规范处理：

```text
口播草案需要画中画方案。
最终交付前需要实际图片或诚实降级状态。
图片生成失败、环境不可用或需要外部工具生成。
用户要求多篇内容并行，且每篇都有图片资产。
final-delivery.html 需要展示、下载或追溯图片。
```

R3 只解决图片资产链，不解决：

```text
R1 单篇主链路自动推进。
R2 parent / child session 和断点恢复。
R4 GitHub 开源包、样例仓库和贡献规范。
```

## 2. 标准资产链

```text
draft
-> static_visual_director_plan
-> visual_plan
-> image_prompt_set
-> image_generation_record
-> image_asset_set
-> image_quality_gate
-> platform_package
-> cover_design_package
-> cover_composition
-> cover_quality_gate
-> html_embed_manifest
-> final_delivery
```

规则：

```text
prompt 卡是设计意图。
static_visual_director_plan 是静态图片编导，不做视频分镜、不做动态 storyboard。
generation_record 是实际生成或降级尝试。
image_asset 是文件和状态。
metadata_sidecar 是 generated 图片的可复现追溯。
platform_package 负责封面标题和平台策略，不产成品封面。
cover_design_package 是 `cover-design-compiler` 生成的封面设计合同，不等同于封面标题。
cover_composition 是叠字、裁切、版式和导出的运行记录。
cover_quality_gate 是封面专项复检，不要求重跑文案质检。
html_embed_manifest 是最终 HTML 怎么展示、下载和链接。
final-delivery.html 不是图片事实源，只是人类验收展示层。
```

图片类型必须先分清：

```text
picture_in_picture_image：正文画中画图片，服务留存、解释、证据、情绪和信任。
cover_image：封面图片，服务点击、推荐流停留、搜索 / 转发第一眼理解。
```

图片生产路径必须先分清：

```text
codex_image2_render：Codex 环境直接调用内置 image 生成图片，写入 assets/images 并嵌入 HTML。
seedream_prompt_delivery：非 Codex 环境不调用外部 API，只交付 Seedream 入参、完整 prompt 和外部生成说明。
manual_upload：人工设计或用户上传。
not_available：无可用路径，必须说明阻断原因。
```

## 3. 内容驱动视觉需求

现行策略固定为：

```text
visual_count_policy=content_derived_unbounded
generation_policy=generate_all_accepted
provider_call_limit=null
cost_gate=not_applicable
accepted_task_dispatch_policy=auto_continue_all_accepted_without_human_confirmation
human_confirmation_required=false
```

先读取受众理解基础和平台观看情境，再拆 `semantic_beats[]`，逐节点建立 `visual_need_candidates[]`。每个候选必须回答 `viewer_problem_without_visual / primary_visual_job / expected_viewer_change / information_added / why_image_is_better_than_talking_head`，并经过 generate / reject 门禁。

合法 `visual_job`：

```text
attention_reset / hook_amplification / concept_explanation / evidence_support
process_demonstration / emotion_amplification / memory_anchor
```

`attention_reset` 不能只因经过几秒触发；情绪任务必须 aligned；证据任务必须 source_bound。纯装饰、重复、high cognitive load、high misleading risk 或无法说明缺图损失的候选必须 reject。

`derived_visual_count=accepted_visual_tasks.length`，允许 0 到 N、不设上限。0 张时必须写 `zero_visual_reason`。Codex 内置 Image 2 对全部 accepted task 生成；不因成本、optional 或 call limit 跳过。封面单独计数。

`visual_need_analysis_status=pass` 时必须写 `generation_dispatch_status=ready_for_prompt_compile / next_skill=image-prompt-compiler` 并自动接续 prompt 编译与 Image 2 生产。H6A 只是不产生图片的分析事务，不是人工确认页；H6B 是紧随其后的生成事务。审美偏好在首轮生成后进入版本化返工，不得成为 accepted task 的预生成门禁。

## 4. 状态边界

`image_status` 只描述单张图片：

```text
generated
pending_external
generation_failed
manual_required
rejected
```

`image_assets_status` 描述整组图片资产：

```text
all_generated
partially_generated
pending_external
generation_failed
manual_required
mixed
not_required
```

状态规则：

```text

generated 必须有 metadata_sidecar_path。
pending_external 只能展示占位、prompt 和插入位置。
generation_failed 必须写 failure_reason 和 retry_suggestion。
manual_required 必须写 human_action_required。
rejected 默认不进最终展示，只保留追溯。
```

## 6. 操作合同

| 操作 | 输入 | 输出 | 状态变化 | 失败处理 |
|---|---|---|---|---|
| `build_static_visual_director_plan` | draft、brief、segment_map | static_visual_director_plan | `director_plan_pass` | 视觉角色不清回 draft / brief |
| `analyze_visual_needs` | draft、brief、audience、platform context、segment_map | visual_need_analysis | `pass / needs_fix / blocked` | 候选缺反事实证明则 reject；关键分析缺失则 needs_fix |
| `build_visual_beats` | draft、brief、visual_need_analysis | beats / accepted_visual_tasks | `visual_beats_ready` | 允许 0 张；不得按时长补图 |
| `build_prompt_set` | beats、visual_language | image_prompt_set | `prompt_set_ready` | prompt_integrity_check fail |
| `decide_image_production_path` | environment_capability、image_task | image_generation_decision / image_production_path | `production_path_ready` | 非 Codex 走 seedream_prompt_delivery |
| `record_generation_attempt` | prompt_card、provider_profile、input_payload | image_generation_record | `generation_record_ready` | 失败也落 generation_failed |
| `materialize_image_asset` | generation_record、provider_output / fallback_status | image_asset | `image_status=*` | 写 fallback_status |
| `write_metadata_sidecar` | image_asset、generation_record、visual_plan | metadata_sidecar | `metadata_sidecar_ready` | generated 缺 sidecar 则 blocked |
| `version_image_asset` | old_asset、new_asset_reason | new image_asset_id | `asset_version_incremented` | 禁止覆盖旧图 |
| `check_image_quality` | image_asset、visual_plan、draft | image_quality_gate | `quality_gate_status` | blocked 不进 HTML |
| `build_cover_design_package` | platform_package、visual_plan、image_asset_set | cover_design_package | `cover_design_pass` | 只有封面标题则 needs_fix |
| `compose_cover_asset` | cover_design_package、cover_background_asset、cover_title、platform_spec | cover_composition / cover_composited_asset | `composition_ready` | 只有底图或文字不可读则 needs_fix |
| `check_static_visual_quality` | image_asset、static_visual_director_plan | static_visual_quality_gate | `static_visual_quality_gate_status` | 低于 7 分回 visual / prompt |
| `check_cover_quality` | cover_design_package、cover_composition、image_asset_set | cover_quality_gate | `cover_quality_gate_status` | fail 回 cover-design-compiler，不默认重跑正文 |
| `check_asset_trace_quality` | image_asset_set、generation_records、metadata | asset_trace_quality_gate | `asset_trace_quality_gate_status` | 缺记录回补 |
| `build_html_embed_manifest` | image_asset_set、delivery_record | html_embed_manifest | `embed_ready` | 缺图显示占位 |

`check_image_quality` 不只检查文件是否存在。真实大循环 `SAMPLE-SESSION-001` 后，至少要记录：

```text
prompt_alignment_score：图片是否符合 prompt，1-10 分。
retention_task_score：图片是否完成留人任务，1-10 分。
content_accuracy_risk：low / medium / high。
usable_in_final_html：yes / no。
review_note：一句人话说明为什么能用或不能用。
```

如果图片存在但 `retention_task_score < 7`，不得作为 L3 样本证据；可以进入本轮交付，但必须在 quality_review 中给 warn 或返工建议。

## 7. 样本模式

```yaml
r3_sample_gate:
  sample_scope: one_content_minimum_asset_chain
  required_outputs:
    - visual_beat
    - prompt_card
    - generation_record
    - image_asset_or_honest_fallback
    - metadata_sidecar_if_generated
    - html_embed_manifest
  pass_condition:
    - no_missing_required_field
    - status_honest
    - local_path_valid_or_pending_clear
    - prompt_traceable
  batch_allowed: true / false
```

样本可以固定验证一张 accepted 图；正式内容必须分析后生成全部 accepted tasks。样本数量不得反推产品配额。

R3 遇到多篇并行时，必须服从 R2：

```text
每个 child session 有独立 image_asset_set。

fan-in 只汇总状态、路径和缺口，不混放图片正文。
parent session 不保存 child 图片正文。
```

## 10. 检查项

R3 编译后的静态检查必须覆盖：

```text
R3CHK-001 visual_need_analysis 存在且策略为 content_derived_unbounded。
R3CHK-002 derived_visual_count 等于 accepted_visual_tasks 数量；允许 0、不设 max。
R3CHK-003 每张 accepted 图有 primary_visual_job、缺图损失和预期观看改变。
R3CHK-004 每张 accepted 图有 insert_after_text / insert_before_text。
R3CHK-005 每张图有完整 prompt_card。
R3CHK-006 每次生成 / 降级有 image_generation_record。
R3CHK-007 generated 图片有 asset_path 且文件存在。
R3CHK-008 generated 图片有 metadata_sidecar_path。
R3CHK-009 pending / failed / manual 不被当成 generated 展示。
R3CHK-010 image_status 和 image_assets_status 不混用。
R3CHK-011 图片重做不覆盖旧 asset。
R3CHK-012 HTML 使用 html_embed_manifest 展示图片或占位。
R3CHK-013 多篇并行时每个 child session 独立 image_asset_set。
R3CHK-014 execution_trace 记录 image_generation / fallback 来源。
R3CHK-015 不调用外部图片 API，不保存 API key。
R3CHK-016 static_visual_director_plan 存在且不是内容语言直译。
R3CHK-017 每张图片有 image_asset_type，且 picture_in_picture_image / cover_image 不混用。
R3CHK-018 每张图片有 image_production_path 和 image_generation_decision。
R3CHK-019 Codex 环境出图写 codex_image2_render，非 Codex 写 seedream_prompt_delivery 和 external_model_payload_path。
R3CHK-020 cover_design_package 存在；不得只用 recommended_cover_title 代替封面设计。
R3CHK-021 cover_quality_gate_status / static_visual_quality_gate_status / asset_trace_quality_gate_status 有明确结果。
R3CHK-022 发布后数据回流不进入本轮 R3 必做门禁，只作为 backlog。
R3CHK-023 cover_background_asset 与 cover_composited_asset 状态分清；只有底图时不得宣称成品封面完成。
R3CHK-024 final-delivery.html 必须展示平台封面策略：共用 / 裁切 / 改标题 / 独立成品封面。
R3CHK-025 使用 model_text_in_image 时必须有文字准确性和小屏可读检查；失败则回 deterministic_overlay / manual_design。
R3CHK-026 cover-design-compiler 是 cover_design_package / cover_composition 的长期 producer；platform-packaging-adapter 只产标题和 cover_variant_set。
R3CHK-027 composition_ready 必须引用实际成品资产；prompt_only 必须有完整 prompt、版式、安全区和人工动作。
R3CHK-028 final-delivery.html 必须区分封面底图、可上传成品、平台变体和 prompt_only，且支持封面局部返工。
```

任一 BLOCKER fail 时，不得宣称 R3 达到 L3 candidate。

## C54-C70 视觉文字与封面质量编译合同

### 执行链

```text
talking-head-image-pip（用户入口）
-> static-visual-director（原子规划）
-> image-prompt-compiler（provider prompt）
-> image-asset-producer（出图 / 叠字 / 降级 / 资产记录）
-> copywriting-quality-review(content_visual_review + visual_text_quality_gate)
-> platform-packaging-adapter
-> cover-design-compiler
-> copywriting-quality-review(cover_review)
-> final-delivery-builder
```

中间三个内部 Skill 不设置用户“继续”门禁。

### 原子规划与三轨

`static_visual_director_plan`、`visual_need_analysis`、`visual_plan.accepted_visual_tasks[]`、`visual_text_plan.visual_text_tasks[]` 固定写入 `intermediate/05-visual-plan.md`。每个 accepted `image_task_id` 必须恰好对应一个 `visual_text_task`。

```text
narration：draft_script_text，负责观点和叙事。
subtitle：可选输入，负责忠实转写。
visual text：负责内心、角色、机制、比较、证据、语境和潜台词。
```

逐图使用 `forbidden / optional / required`。forbidden 的 units 必须为空；required 至少有一个通过预算、信息增量和来源检查的 unit。

### 文字预算与来源

```text
role / comparison / context：2-8 个汉字。
inner_voice / subtext / visual_narration：4-14 个汉字。
mechanism：每节点 2-6 个汉字，默认最多 4 个节点。
source_note：6-24 个汉字或短来源名。
单图默认 1-3 个 unit；证据图可额外增加 1 个 source_note。
```

超出预算必须写 `overflow_reason`。含字任务的 semantic delta、glance comprehension、mobile readability 默认均不得低于 7。

证据、数据、引语、截图和事实标签的每个文字单元必须有：

```text
is_source_required=true
evidence_source_type
evidence_source_id
evidence_source_path
source_binding_status=source_bound
```

生成情境图不得承担 `evidence_support`。source-bound 要求 type / id / path 同时可解析。

### 渲染、质量和封面

```text
no_text：禁止含字。
model_text_in_image：只用于短字候选，必须检查准确性。
deterministic_overlay：逐字准确默认路径。
source_native_text：保留原截图 / 证据文字。
prompt_only：非 Codex 环境交付完整 prompt、exact text、placement 和 human action。
```
