# R3 图片资产执行规范

> 状态：active  
> 所属路线：R3 画中画与图片资产模型  
> 主责：把 R3-C01 到 R3-C25 编译成运行时可执行规范，供 `talking-head-image-pip`、`copywriting-quality-review`、`final-delivery-builder` 和后续 dry-run 使用。  
> 边界：本文件不实现外部图片 API、不调用 Seedream、不建设图片服务、不自动发布。

---

## 1. 适用场景

当工作流进入以下场景时，必须按本规范处理：

```text
口播草案需要画中画方案。
最终交付前需要实际图片或诚实降级状态。
图片生成失败、环境不可用或需要外部工具生成。
用户要求多篇内容并行，且每篇都有图片资产。
final-delivery.html 需要展示、下载或追溯图片。
```

R3 只解决图片资产链，不解决：

```text
R1 单篇主链路自动推进。
R2 parent / child session 和断点恢复。
R4 GitHub 开源包、样例仓库和贡献规范。
```

---

## 2. 标准资产链

```text
draft
-> visual_plan
-> image_prompt_set
-> image_generation_record
-> image_asset_set
-> image_quality_gate
-> html_embed_manifest
-> final_delivery
```

规则：

```text
prompt 卡是设计意图。
generation_record 是实际生成或降级尝试。
image_asset 是文件和状态。
metadata_sidecar 是 generated 图片的可复现追溯。
html_embed_manifest 是最终 HTML 怎么展示、下载和链接。
final-delivery.html 不是图片事实源，只是人类验收展示层。
```

---

## 3. 图片数量预算

默认视觉预算：

| 内容长度 / 结构 | required_visuals | optional_visuals |
|---|---:|---:|
| 30 秒以内，单观点 | 1 | 1 |
| 30-60 秒，标准口播 | 2 | 1-2 |
| 60-90 秒，信息密度较高 | 3 | 1-2 |
| 超 90 秒或多段结构 | 3-4 | 2 |

增加图片的条件：

```text
前 5 秒需要停住划走。
口播出现抽象概念，缺图会听不懂。
需要证据、可信度或产品能力说明。
转场信息密度下降，需要节奏打断。
```

减少图片的条件：

```text
图片只是好看但不承担留存任务。
图片抢口播注意力。
图片暗示未实现能力。
图片和热点桥接太硬。
同类视觉重复。
```

## 4. 状态边界

`image_status` 只描述单张图片：

```text
generated
pending_external
generation_failed
manual_required
rejected
```

`image_assets_status` 描述整组图片资产：

```text
all_generated
partially_generated
pending_external
generation_failed
manual_required
mixed
not_required
```

状态规则：

```text
generated 必须有本地 asset_path，且文件可读。
generated 必须有 metadata_sidecar_path。
pending_external 只能展示占位、prompt 和插入位置。
generation_failed 必须写 failure_reason 和 retry_suggestion。
manual_required 必须写 human_action_required。
rejected 默认不进最终展示，只保留追溯。
```

## 6. 操作合同

| 操作 | 输入 | 输出 | 状态变化 | 失败处理 |
|---|---|---|---|---|
| `plan_visual_budget` | draft、duration、segment_map | visual_budget | `visual_plan_active` | 无法估算时按 30-60 秒规则 |
| `build_visual_beats` | draft、brief、retention risk | beats | `visual_beats_ready` | 首屏无任务则回 draft |
| `build_prompt_set` | beats、visual_language | image_prompt_set | `prompt_set_ready` | prompt_integrity_check fail |
| `record_generation_attempt` | prompt_card、provider_profile、input_payload | image_generation_record | `generation_record_ready` | 失败也落 generation_failed |
| `materialize_image_asset` | generation_record、provider_output / fallback_status | image_asset | `image_status=*` | 写 fallback_status |
| `write_metadata_sidecar` | image_asset、generation_record、visual_plan | metadata_sidecar | `metadata_sidecar_ready` | generated 缺 sidecar 则 blocked |
| `version_image_asset` | old_asset、new_asset_reason | new image_asset_id | `asset_version_incremented` | 禁止覆盖旧图 |
| `check_image_quality` | image_asset、visual_plan、draft | image_quality_gate | `quality_gate_status` | blocked 不进 HTML |
| `build_html_embed_manifest` | image_asset_set、delivery_record | html_embed_manifest | `embed_ready` | 缺图显示占位 |

`check_image_quality` 不只检查文件是否存在。真实大循环 `S20260707-001` 后，至少要记录：

```text
prompt_alignment_score：图片是否符合 prompt，1-10 分。
retention_task_score：图片是否完成留人任务，1-10 分。
content_accuracy_risk：low / medium / high。
usable_in_final_html：yes / no。
review_note：一句人话说明为什么能用或不能用。
```

如果图片存在但 `retention_task_score < 7`，不得作为 L3 样本证据；可以进入本轮交付，但必须在 quality_review 中给 warn 或返工建议。

## 7. 样本模式

R3 图片资产链先验证单篇最小图片资产链：

```yaml
r3_sample_gate:
  sample_scope: one_content_minimum_asset_chain
  required_outputs:
    - visual_beat
    - prompt_card
    - generation_record
    - image_asset_or_honest_fallback
    - metadata_sidecar_if_generated
    - html_embed_manifest
  pass_condition:
    - no_missing_required_field
    - status_honest
    - local_path_valid_or_pending_clear
    - prompt_traceable
  batch_allowed: true / false
```

样本可以只验证一张 required 图；正式内容仍按视觉预算生成 required_visuals。

## 8. 多篇隔离

R3 遇到多篇并行时，必须服从 R2：

```text
每个 child session 有独立 image_asset_set。
每个 child session 有独立 assets/images/。
fan-in 只汇总状态、路径和缺口，不混放图片正文。
parent session 不保存 child 图片正文。
```

## 9. 最终 HTML 嵌入规则

`html_embed_manifest` 必须包含：

```text
embed_id
image_asset_id
display_mode
display_path
download_path
source_prompt_path
metadata_sidecar_path
generation_record_path
insert_after_text
insert_before_text
status_label
human_note
```

展示规则：

```text
generated：展示图片预览和下载。
pending_external：展示占位、可复制 prompt、插入位置。
generation_failed：展示失败原因和重试建议。
manual_required：展示人工设计任务。
rejected：默认不展示，只在追溯区保留。
外部链接图片：只能作为 reference_note；必须落本地资产后才能 generated。
```

## 10. 检查项

R3 编译后的静态检查必须覆盖：

```text
R3CHK-001 visual_budget 存在。
R3CHK-002 required_visuals 数量符合预算或有增减理由。
R3CHK-003 每张 required 图有 retention_task。
R3CHK-004 每张图有 insert_after_text / insert_before_text。
R3CHK-005 每张图有完整 prompt_card。
R3CHK-006 每次生成 / 降级有 image_generation_record。
R3CHK-007 generated 图片有 asset_path 且文件存在。
R3CHK-008 generated 图片有 metadata_sidecar_path。
R3CHK-009 pending / failed / manual 不被当成 generated 展示。
R3CHK-010 image_status 和 image_assets_status 不混用。
R3CHK-011 图片重做不覆盖旧 asset。
R3CHK-012 HTML 使用 html_embed_manifest 展示图片或占位。
R3CHK-013 多篇并行时每个 child session 独立 image_asset_set。
R3CHK-014 execution_trace 记录 image_generation / fallback 来源。
R3CHK-015 不调用外部图片 API，不保存 API key。
```

任一 BLOCKER fail 时，不得宣称 R3 达到 L3 candidate。
