# R1 人类门禁决策枚举与恢复规则

> 状态：已确认并已编译引用  
> 所属路线：R1 方法论没有编译成执行合同  
> 解决缺口：把“什么时候停”补全为“用户怎么回复、状态怎么变、从哪里恢复”。  
> 边界：本文件只定义 R1 产品层门禁枚举；不改 `skills/*/SKILL.md`，不替代 `docs/reference/人类引导与任务后导航规范.md`。

---

## 1. 为什么要补门禁决策枚举

已有规则已经说明：

```text
能自动衔接的，不向用户索要“继续”。
真正需要人判断的，才停给用户。
```

但成熟 workflow 还需要回答：

```text
用户回复“认可”后，哪个状态变成什么？
用户回复“重找一轮”后，回到哪里？
用户回复“只改抖音标题”后，算返工还是最终确认？
用户回复“三篇都做”是 R1 能处理，还是交给 R2 fan-out？
```

R1 的目标不是解决全部多分支，而是把每个人类门禁的决策出口定义清楚，避免 agent 临场猜。

---

## 2. 决策类型

R1 使用以下统一决策类型：

| decision_type | 含义 | 是否继续自动推进 |
|---|---|---|
| `approve` | 用户认可当前推荐或当前产物 | 是 |
| `select` | 用户从候选中选择一个明确对象 | 是 |
| `edit` | 用户要求局部修改 | 回对应 artifact 后再自动推进 |
| `reject` | 用户否定当前产物 | 回上游或重做 |
| `archive` | 用户决定本条暂不继续 | 不继续，进入归档 |
| `export` | 用户要求生成转交包或单文件 | 进入导出链路 |
| `publish_record` | 用户已人工发布，要求记录 | 只记录，不自动发布 |
| `cancel` | 用户取消当前链路 | 停止本 session |
| `branch_request` | 用户要求多篇、多账号、旁支任务 | R1 只识别并交给 R2 规则，不在同一链路硬跑 |

禁止把“继续”作为必需 decision_type。  
用户不需要说继续，workflow 只在真实门禁处等待上述决策。

## 3. 门禁枚举表

### 3.1 账号档案缺失或不完整

```yaml
gate_id: account_profile_missing_or_incomplete
owner_skill: propagation-router
allowed_decisions:
  - edit
  - cancel
state_updates:
  edit:
    target_artifact: account_profile
    profile_status: profile_needs_info / profile_ready
    auto_next_after_reply: account_profile_quality_check
  cancel:
    session_status: session_archived
    last_decision: cancel
```

人话引导重点：

```text
一次最多问 3 个问题，用户回答后由 AI 归纳，不把原话堆进档案。
```

### 3.2 换账号后档案确认

```yaml
gate_id: account_profile_reconfirm_after_switch
owner_skill: propagation-router
allowed_decisions:
  - approve
  - edit
  - cancel
state_updates:
  approve:
    account_profile_confirmed_for_session: yes
    next_skill: product_profile_check
  edit:
    target_artifact: account_profile
    next_skill: account_profile_quality_check
  cancel:
    session_status: session_archived
    last_decision: cancel
```

用户常见回复映射：

| 用户回复 | decision_type |
|---|---|
| 认可 / 同意 / 没变化 / 就按这个 | `approve` |
| 目标人群改一下 / 禁区加一条 | `edit` |
| 先不做这个账号 | `cancel` |

### 3.3 产品 / 活动对象不清

```yaml
gate_id: product_or_campaign_unclear
owner_skill: propagation-router
allowed_decisions:
  - select
  - edit
  - cancel
state_updates:
  select:
    product_profile_id_or_campaign_profile_id: selected
    next_skill: hotspot-topic-research
  edit:
    target_artifact: product_profile / campaign_profile
    next_skill: product_profile_check
  cancel:
    session_status: session_archived
    last_decision: cancel
```

### 3.4 Topic Gate

```yaml
gate_id: topic_gate
owner_skill: hotspot-topic-research
allowed_decisions:
  - select
  - reject
  - branch_request
  - cancel
state_updates:
  select:
    topic_status: topic_selected_for_brief
    next_skill: content-brief-compiler
  reject:
    topic_status: topic_rejected
    next_skill: hotspot-topic-research
  branch_request:
    last_decision: branch_request
    blocked_reason: needs_r2_fan_out
    session_status: session_waiting_human
    next_skill: propagation-router
  cancel:
    session_status: session_archived
    last_decision: cancel
```

R1 限制：

```text
用户选择单个 topic_id，R1 自动继续。
用户说“三篇都做”，R1 只能识别为 branch_request；真正 fan-out / fan-in 进入 R2，不在一个 session 里硬跑三篇。
```

### 3.5 Brief 阻断

```yaml
gate_id: brief_blocked
owner_skill: content-brief-compiler
allowed_decisions:
  - edit
  - reject
  - cancel
state_updates:
  edit:
    target_artifact: content_brief / topic_card / product_profile
    next_skill: content-brief-compiler
  reject:
    target_artifact: topic_card
    next_skill: hotspot-topic-research
  cancel:
    session_status: session_archived
    last_decision: cancel
```

Brief 通过不是门禁，不要求用户回复。

### 3.6 Hook / 文案策略取舍

```yaml
gate_id: hook_choice
owner_skill: copywriting-draft-writer
allowed_decisions:
  - select
  - edit
state_updates:
  select:
    selected_hook_route: selected
    draft_status: draft_created
    next_skill: talking-head-image-pip
  edit:
    target_artifact: draft
    next_skill: copywriting-draft-writer
```

适用条件：

```text
只有当 A/B/C 方案存在真实策略差异，且默认推荐不足以自动决定时才停。
```

### 3.7 Draft 阻断

```yaml
gate_id: draft_blocked
owner_skill: copywriting-draft-writer
allowed_decisions:
  - edit
  - reject
  - cancel
state_updates:
  edit:
    target_artifact: draft
    next_skill: copywriting-draft-writer
  reject:
    target_artifact: content_brief
    next_skill: content-brief-compiler
  cancel:
    session_status: session_archived
    last_decision: cancel
```

阻断原因包括：

```text
hook_score < 7
body_information_density_score < 7
content_promise 无法兑现
core_mechanism 不清
```

### 3.8 视觉策略取舍 / 图片不可用

```yaml
gate_id: visual_strategy_choice
owner_skill: talking-head-image-pip
allowed_decisions:
  - select
  - edit
state_updates:
  select:
    visual_plan_status: visual_plan_pass
    next_skill: copywriting-quality-review
  edit:
    target_artifact: visual_plan
    next_skill: talking-head-image-pip
```

```yaml
gate_id: image_generation_unavailable
owner_skill: talking-head-image-pip
allowed_decisions:
  - approve
  - edit
state_updates:
  approve:
    image_assets_status: pending_external / manual_required
    next_skill: copywriting-quality-review
  edit:
    target_artifact: image_prompt_set / visual_plan
    next_skill: talking-head-image-pip
```

规则：

```text
缺图可以降级，但必须记录 image_assets_status。
不得把 pending_external 说成 generated。
```

```yaml
allowed_decisions:
  - edit
  - cancel

```

### 3.11 最终 HTML 验收

```yaml
gate_id: final_delivery_review
owner_skill: final-delivery-builder
allowed_decisions:
  - approve
  - edit
  - archive
  - export
  - publish_record
  - cancel
state_updates:
  approve:
    delivery_status: delivery_confirmed
    approval_status: approval_approved
    publish_status: publish_not_started
    session_status: session_waiting_human
  edit:
    approval_status: approval_needs_revision
    next_skill: selected_by_back_to_target
  archive:
    delivery_status: delivery_archived
    approval_status: approval_not_required
    session_status: session_archived
  export:
    last_decision: export
    final_delivery_status: needs_export
    next_skill: final-delivery-builder
  publish_record:
    publish_status: publish_manually_published / publish_unknown
    next_skill: workflow_session_record
  cancel:
    delivery_status: delivery_discarded
    session_status: session_archived
    last_decision: cancel
```

局部返工映射：

| 用户回复 | target_artifact | next_skill |
|---|---|---|
| 只改抖音标题 | `platform_package` | `platform-packaging-adapter` |
| 回到口播改前 5 秒 | `draft` | `copywriting-draft-writer` |
| 回到画中画改首屏图 | `visual_plan` | `talking-head-image-pip` |
| 核心观点不对 | `content_brief` | `content-brief-compiler` |
| 这个选题不要了 | `topic_card` | `hotspot-topic-research` |

## 4. 禁止门禁

以下节点不得设置人类门禁：

| 节点 | 正确动作 |
|---|---|
| Brief 通过 | 自动进入 draft |
| draft 通过最低门槛 | 自动进入 visual_plan |
| visual_plan 通过 | 自动进入 quality_review |
| review_pass | 自动进入 platform_package |
| platform_package 完成 | 自动进入 final_delivery |
| 用户明确选了单个 topic_id | 自动进入 content_brief |

## 5. 任务后导航输出

每个人类门禁都必须输出：

```yaml
human_prompt:
recommended_action:
human_reply_examples:
decision_mapping:
auto_next_after_reply:
state_updates:
```

其中 `decision_mapping` 是 R1 编译时必须新增或吸收的关键字段。它把用户自然语言映射成结构化 `decision_type`。
