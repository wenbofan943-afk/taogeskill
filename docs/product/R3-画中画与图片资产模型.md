# R3 画中画与图片资产模型

> 状态：r3_confirmed_compiled_pending_external_dry_run_pass  
> 所属路线：R3 画中画与图片资产模型  
> 主责：定义从视觉策略到图片资产、质检、HTML 嵌入、外部模型降级的字段和动作。  
> 边界：本文件是 R3 产品真源之一，已进入规则 / skill 编译并通过 pending_external dry-run；不实现 API、不调用外部图片模型、不把 pending_external 写成 generated。

---

## 1. 核心对象

| 对象 | 技术名 | 位置 | 说明 |
|---|---|---|---|
| 视觉方案 | `visual_plan` | `intermediate/05-visual-plan.md` | 从 draft 拆出的画中画策略 |
| 视觉 beat | `visual_beat` | `visual_plan.beats[]` | 一段口播对应的视觉任务 |
| 提示词组 | `image_prompt_set` | `visual_plan.image_prompts[]` | 完整 prompt 卡集合 |
| 图片任务 | `image_task` | `visual_plan.required_visuals[]` | 一张图的生成任务 |
| 生成记录 | `image_generation_record` | `assets/images/generation-records/` | 一次 provider / 模型 / 参数 / 输出 / 失败尝试 |
| 图片资产组 | `image_asset_set` | `assets/images/image-assets.md` | 图片文件和状态总表 |
| 单张图片资产 | `image_asset` | `image_asset_set.assets[]` | 图片路径、状态、provider、prompt_used |
| 元数据 sidecar | `image_metadata_sidecar` | `assets/images/metadata/` | 图片外置可复现元数据，补足图片内嵌 metadata 不可靠的问题 |
| HTML 嵌入记录 | `html_embed_manifest` | `deliverables/content-delivery-record.md` | 最终 HTML 展示 / 下载 / 追溯配置 |

## 2. 标准流程

```text
copywriting-draft-writer
-> talking-head-image-pip 生成 visual_plan
-> image_asset_generation 生成 generation_record / metadata_sidecar 或 fallback_status
-> copywriting-quality-review 做文案 + 视觉联合质检
-> platform-packaging-adapter 汇总发布物料
-> final-delivery-builder 嵌入实际图片或降级占位
```

自动推进口径：

```text
visual_plan_status=visual_plan_pass 且 prompt_integrity_check 全部 pass -> 进入 quality_review。
最终交付前 required image 必须 generated 或有明确 fallback 状态。
图片缺失不阻断无图 HTML，但必须在 HTML 中清楚显示。
样本模式未通过前，不进入多篇批量出图。
```

状态边界：

```text
`image_status` 只用于单张图片资产：generated / pending_external / generation_failed / manual_required / rejected。
`image_assets_status` 用于整组图片资产：all_generated / partially_generated / pending_external / generation_failed / manual_required / mixed / not_required。
```

## 4. visual_beat 字段

```yaml
visual_beat:
  beat_id:
  segment_type:
  script_range:
    insert_after_text:
    insert_before_text:
  estimated_time:
  segment_function:
  retention_risk:
  visual_need: required / optional / not_needed / not_recommended
  retention_task:
  visual_type:
  visual_one_liner:
  no_visual_loss:
  risk_notes:
  image_task_id:
```

规则：

```text
visual_need=required / optional 才能进入 image_task。
visual_need=not_needed / not_recommended 不生成 prompt，除非用户明确要求。
每个 required beat 只能有一个主 retention_task。
```

## 7. image_generation_record 字段

每次图片生成或外部待生成都必须写生成记录。即使当前环境不能直接出图，也要写清楚为什么没有生成。

```yaml
image_generation_record:
  generation_run_id:
  generation_attempt_id:
  image_task_id:
  source_prompt_id:
  provider:
  model:
  provider_mode: codex_builtin / external_api / manual_upload / not_available
  input_schema_version:
  input_payload_path:
  prompt_used:
  negative_prompt:
  aspect_ratio:
  width:
  height:
  seed:
  reference_image_paths:
  started_at:
  finished_at:
  generation_status: generated / pending_external / generation_failed / manual_required
  provider_response_id:
  output_asset_path:
  failure_reason:
  retry_suggestion:
  execution_trace_ref:
```

记录原则：

```text
prompt 卡是“设计意图”，generation_record 是“实际尝试”。
实际送给 provider 的 prompt、尺寸、负面词和参考图必须能从 input_payload_path 找到。
失败不是空白，失败也必须成为可恢复记录。
```

## 8. image_metadata_sidecar 字段

图片文件自身可能丢失 metadata，或不同 provider 的 metadata 格式不一致。因此 R3 要求每张 generated 图片旁边保留 sidecar。

```yaml
image_metadata_sidecar:
  image_asset_id:
  asset_path:
  source_prompt_id:
  generation_run_id:
  provider:
  model:
  input_schema_version:
  prompt_used_full_path:
  generation_record_path:
  visual_plan_path:
  quality_gate_path:
  created_at:
  checksum:
  license_note:
  privacy_note:
```

sidecar 最小要求：

```text
图片可预览是一回事，图片可追溯是另一回事。
最终 HTML 链接回 sidecar，而不是只链接回图片文件。
```

## 9. image_quality_gate

图片进入最终交付前，必须通过以下检查：

| 检查项 | 阻断条件 |
|---|---|
| 文件存在 | `generated` 但 asset_path 不存在 |
| 小屏可读 | 手机竖屏无法看懂主体 |
| 口播贴合 | 图片和 insert 位置语义不一致 |
| 产品边界 | 暗示未实现能力、后台数据、私信、抓取隐私 |
| 版权 / 隐私 | 出现真实个人信息、车牌、手机号、微信号 |
| prompt 追溯 | 找不到 source_prompt_id 或 prompt_used |
| 生成记录 | 找不到 generation_run_id；`generated` 图片找不到 metadata_sidecar_path；pending / failed / manual 找不到生成记录或人工任务说明 |
| 状态诚实 | pending_external 被当成 generated 展示 |
| 本地资源 | HTML 需要展示的图不是 session 本地文件 |

输出：

```yaml
image_quality_gate:
  image_asset_id:
  quality_gate_status: pass / needs_fix / blocked
  blocking_issues:
  suggestions:
  checked_at:
```

## 10. html_embed_manifest

最终 HTML 嵌入图片时必须有：

```yaml
html_embed_manifest:
  final_delivery_id:
  image_asset_set_id:
  embeds:
    - embed_id:
      image_asset_id:
      display_mode: image / placeholder / prompt_card / rejected_hidden
      display_path:
      download_path:
      source_prompt_path:
      metadata_sidecar_path:
      generation_record_path:
      insert_after_text:
      insert_before_text:
      status_label:
      human_note:
```

展示规则：

```text
generated：展示图片预览和下载。
pending_external：展示占位、可复制 prompt、插入位置。
generation_failed：展示失败原因和重试建议。
manual_required：展示人工设计任务。
rejected：默认不展示，只在追溯区保留。
外部链接图片：不展示为最终图，只能作为 reference_note；必须落本地资产后才能 generated。
```

## 12. 样本模式与批量边界

R3 采用 dbskill 的“先样本、再批量”思想。即使用户要求多篇一起做，图片资产链也必须先在单篇样本上跑通。

```yaml
r3_sample_gate:
  sample_scope: one_content_minimum_asset_chain
  required_outputs:
    - visual_beat
    - prompt_card
    - generation_record
    - image_asset_or_honest_fallback
    - metadata_sidecar_if_generated
    - html_embed_manifest
  pass_condition:
    - no_missing_required_field
    - status_honest
    - local_path_valid_or_pending_clear
    - prompt_traceable
  batch_allowed: true / false
```

批量限制：

```text
样本不过，不开多篇出图。
样本可以只验证 1 张 required 图的最小资产链；正式内容仍按默认视觉预算生成 required_visuals。
多篇任务必须按 R2 child session 隔离 image_asset_set。
fan-in 只汇总状态和路径，不混放不同账号或不同作品的图片。
```

## 13. R3 操作合同

| 操作 | 输入 | 输出 | 状态变化 | 失败处理 |
|---|---|---|---|---|
| `plan_visual_budget` | draft、duration、segment_map | visual_budget | `visual_plan_active` | 无法估算时用标准 30-60 秒规则 |
| `build_visual_beats` | draft、brief、retention risk | beats | `visual_beats_ready` | 首屏无任务则回 draft |
| `build_prompt_set` | beats、visual_language | image_prompt_set | `prompt_set_ready` | prompt_integrity_check fail |
| `record_generation_attempt` | prompt_card、provider_profile、input_payload | image_generation_record | `generation_record_ready` | 失败也落 generation_failed |
| `materialize_image_asset` | generation_record、provider_output / fallback_status | image_asset | `image_status=generated / pending_external / generation_failed / manual_required` | 写 fallback_status |
| `write_metadata_sidecar` | image_asset、generation_record、visual_plan | metadata_sidecar | `metadata_sidecar_ready` | generated 缺 sidecar 则 blocked |
| `version_image_asset` | old_asset、new_asset_reason | new image_asset_id | `asset_version_incremented` | 禁止覆盖旧图 |
| `check_image_quality` | image_asset、visual_plan、draft | image_quality_gate | `quality_gate_status` | blocked 不进 HTML |
| `build_html_embed_manifest` | image_asset_set、delivery_record | html_embed_manifest | `embed_ready` | 缺图显示占位 |

## 14. R3 验收口径

R3 产品层通过标准：

```text
图片数量不是看心情，有默认预算和增减规则。
每张 required 图都有 retention_task、insert_position、prompt_card、provider_profile、image_status。
generated / pending_external / generation_failed / manual_required / rejected 状态不会混用。
每次生成尝试都有 generation_record。
generated 图片有本地 asset_path 和 metadata_sidecar_path。
重做图片不会覆盖旧资产。
样本模式通过前不进入批量图片生产。
final_delivery.html 知道每张图怎么展示、下载和追溯。
外部模型降级只保留兼容字段，不误称已实现。
```

## 15. 对标后返修结论

```text
R3 必须把图片资产当成“可恢复运行记录”，不是简单附件。
R3 必须区分 prompt_card、generation_record、image_asset、metadata_sidecar、html_embed_manifest。
R3 必须把样本模式写入产品层，否则多篇并行时图片链最容易串台。
R3 已确认，并已进入字段词典、talking-head-image-pip、final-delivery-builder 和执行透明度规范编译；下一步只补 generated 图片路径样本，不扩大成完整真实测试。
```
