| R7-C01 | R7 定位为单篇内容的语义工作流总控与交付编译层，不重做 R1 / R3 / R5 / R6 的内容判断。 |

| R7-C04 | “自主完成”不等于没有 AI；Codex 只按已注册合同执行、没有临场补流程 / 补字段 / 手改机器对象时，可以计入自主完成。 |
| R7-C05 | P0 plan / event / projection 继续是运行事实源；R7 不另建第二套 workflow state。 |
| R7-C06 | 先编译直供稿全链，再接热点稿入口；两者保留各自正确的结构 / 草稿前置顺序，在 current draft、current structure plan、current structure-bound beat map 闭合后共用口播质检、视觉、包装和交付链。 |
| R7-C07 | `workflow_blueprint` 版本化登记每个 node 的 Skill、输入选择、输出、状态、路由、重试和 stale 范围；`contract_status_registry` 统一登记每个合同的 active / superseded / compiled 状态。 |
| R7-C08 | coordinator 必须从 projection 得到唯一下一步；未声明并行时不能让 Codex 自选步骤。 |
| R7-C09 | 每个语义步骤先生成 `semantic_task_envelope`，明确业务目标、输入 ID / hash、Skill、schema、allowed statuses / actions 和恢复上下文。 |
| R7-C10 | Codex 只提交 typed `semantic_artifact_submission`；不得直接写 current pointer、event、projection 或最终 candidate。 |
| R7-C11 | deterministic submitter 独占 schema、enum、cross-reference、digest、revision、lineage、pointer-last 和 event / projection 提交。 |
| R7-C12 | submission 失败保留证据并新建 revision；禁止直接修改已提交 revision、event、projection 或 current pointer。 |

| R7-C15 | candidate compiler 在最终 runtime 前执行 task、producer、cross-object、asset、candidate、render 六级预检，并把错误路由回 owning layer。 |

| R7-C17 | 每个 materialized 封面 rendition 必须绑定同 rendition、同 revision、同 output / preview hash 的真实 raster review，缺失时在 candidate 前阻断。 |
| R7-C18 | action code 只能来自版本化 registry；用户可读文案与 code 分开，未注册自然语言不做相似猜测。 |

| R7-C22 | 截图按 CSS pixel 作为复现证据；布局结论直接来自页面测量，截图仍需人工观察实际封面、PIP 和文字层级。 |
| R7-C23 | 普通内容运行缺浏览器能力时为 `not_tested + ready_with_warnings`；template / renderer 变更或 public release 缺真实视口证据时阻断。 |

| R7-C28 | 编译按 H1 blueprint / registry、H2 coordinator / submitter、H3 直供 producer adapters、H4 candidate compiler、H5 viewport / transparency、H6 热点入口与真实回归六批推进。 |
