## 已确认

### R6-C01 直供文案是一等入口

用户自己写好的文案可以直接进入内容生产，不必先伪造热点研究或热点选题。

### R6-C02 直供不等于跳过工作流

直供文案跳过的是 R5 热点发现 / 候选选择；仍需账号启动、Brief、主张地图、视觉、质检、平台包装、封面、最终交付、trace 和 checkpoint。

### R6-C03 四种直供意图

`direct_delivery`、`direct_polish`、`direct_evidence_enrichment`、`direct_to_radar` 由用户自然语言选择；默认不得把它们混为“自动找热点重写”。

### R6-C04 原稿保护与改写口径

用户未说明是否可重写时，默认保留其声音，只做必要风险 / 事实提示；结构性重写前才问一次人话确认。

### R6-C05 直供内容对象

新增 `direct_content_intake` 和 `direct_content_card`，记录原稿 digest、创作意图、改写边界、主张地图和 `topic_origin=direct_user_input`；它们是 R1 的合法 producer，不伪造 `topic_card`。

### R6-C06 HTML 不等于完成

只有完整交付链、状态投影、trace、checkpoint 和终态 finalize 闭合，才能写 `session_completed`；单独生成 HTML 只能是 `html_ready` / preview。

### R6-C07 证据截图只为具体主张服务

只对选择后的 `factual_claim / quote / statistic` 制定检索计划；不对每句情绪、个人经验或预测强行检索。

### R6-C08 五态证据判断

`supported / refuted / not_enough_info / contested / not_checked` 是主张与来源的关系；没有证据是正常结果，不得硬凑截图。

### R6-C09 四层事实分离

页面可访问、截图完整、来源支持主张、当前表达可发布四件事分别记录，不用“置信度”一个数混写。

### R6-C10 截图是来源型资产

新闻 / 数据 / 公告 / 引语截图一律 `image_production_path=source_capture`，具备 URL、发布者、时间、捕获时间、片段定位、hash 和 binding；不调用 Image 2。

### R6-C11 来源与解读必须分层

画中画展示来源条与“涛哥解读”条；创作者推论不得排版成来源原话，来源截图不得被剪裁到无法识别出处。

### R6-C12 最小必要与可追溯

默认只裁切支撑主张的必要范围；不自动截取登录 / 付费墙 / 私密内容、评论区头像或个人资料。视频可用短来源标签，交付页保留完整追溯信息。

### R6-C13 版权 / 隐私不靠免责声明绕过

“内容来自网络”不构成可用判断。权利或隐私风险未通过时，降级为文字来源卡 / 不使用截图 / 请求授权材料。

### R6-C14 截图外部副作用的恢复

捕获后先落 attempt、outcome、输出路径和 hash；裁切 / 叠字被中断时先 reconcile 已有输出，禁止盲目重抓。

### R6-C15 首期不建采集器或归档 SaaS

首期只完成按需、公开页面、单一证据截图的本地生产合同；不自动登录、不批量采集、不建后台。WARC 网页归档留作后续高风险 / 长期引用能力。

### R6-C16 R5 / R3 的职责不串位

R5 负责发现、事件归并和热点来源；R6 允许直供文案按需查询具体主张；R3 负责逐图价值、视觉文字和资产质量。三者通过 IDs / bindings 衔接，不相互伪造上游。

### R6-C17 账号视觉身份实际生效

证据事实、涛哥解读、风险提示和来源条采用账号的 `evidence_visual_grammar`，但不将账号风格用于掩盖来源或改变事实状态。

### R6-C18 编译六层闭合

字段词典、Skill / CONTRACT、机器 Schema / runtime、正反 fixture、专项 checker、最终 HTML / renderer 必须同步；仅改产品说明不算实现。

### R6-C19 真实运行边界

真实账号回归只在私有生产区，先使用公开可访问来源；不登录、不自动发布、不把真实截图、正文、prompt 或账号资料放入公开 Git。

### 推荐确认口径

```text
一篇稿先完整拆成观众理解会发生变化的内容节点，再做口播和视觉。
不套固定时长、固定结构和万能分数；问题必须给出精确原句和理由，但修改建议不自动阻止继续生产。
用户直供稿默认保留原声，改变立场、事实强度或核心结构时再确认。
直供稿和热点稿采用同一质量标准，但各自保留真实来源身份。
```

### 编译前二义性审核结论

产品真源已给出以下唯一口径：

```text
结构 plan ≠ beat map ≠ segment map ≠ visual coverage。
热点新写与用户直供采用不同前置顺序，汇合后读取同一 current revision 链。
直供稿先有 semantic-only 节点图用于诊断，结构选定后再生成 structure-bound 节点图；不形成循环依赖。
问题状态、结构 / 脚本就绪和视觉素材就绪分别判断。
等待人类授权、等待素材与真正合同阻断分别判断；正常等待不能写成失败。
图片需要数、已成素材数、Image 2 任务数、来源截图数、插入次数、平台 rendition 与封面分别计数。
复用历史非证据素材会建立本稿 current task；历史来源截图继续保持证据 binding；复用本稿已有 task 只增加覆盖引用 / occurrence。
机器只判覆盖、字段和引用；内容 / 视觉质量必须由 Codex 或人实际审查并写理由。
immutable revision 写完、校验、投影和 lineage 闭合后，最后更新 current pointer。
```

完整条件必填、状态枚举、数量公式、current pointer 写入顺序和 v0.1 / v0.4 / v0.5 兼容矩阵见 [R6 口播脚本与视觉协同编排第 12 节](./R6-口播脚本与视觉协同编排.md#12-编译前二义性反审计)。
