## 产品二义性审核

| 问题 | 结论 |
|---|---|
| “拆 Skill”是否等于新增更多用户命令？ | 否。只有 router 内部两个人类门禁改变 owner，用户自然语言入口不变。 |
| references 是否每次都读？ | 否。每个 reference 必须有版本化或状态化条件。 |
| 行数下降是否自动等于效果提升？ | 否。必须通过旧版 / 新版 A/B。 |
| 是否重做热点产品逻辑？ | 否。R5 业务决定不重开。 |
| 是否把四个平台拆成四条 workflow？ | 否。仍形成一个 platform package。 |
| 是否改变历史 session？ | 否。历史 session 只按旧 baseline replay。 |
| 是否包含 Codex 自带 Skill？ | 否。只包含项目业务 Skill。 |
| 是否已经完成编译？ | H1 机器清单、H2 热点 Skill 收缩和 H3 router / 内部人工门禁已完成；平台包装仍待 H4，A/B 与总回归仍待 H5。 |
