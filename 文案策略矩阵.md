# 文案策略矩阵

> 状态：传播研究方法论单元  
> 主责：定义公开互动分析工具自媒体传播的内容策略类型、目标、指标、风险和适用场景。  
> 边界：本文只指导传播和文案规划，不是产品验收包，不是工程开发任务，不直接改变产品功能或发版链路。

---

## 一、核心判断

本项目传播不是只有一种文案。

不同内容承担不同任务：

```text
有人负责破圈
有人负责获客
有人负责建立信任
有人负责转化试用
有人负责复盘共创
```

因此，热点进入文案前，必须先判断：

```text
这条内容属于哪种文案策略？
```

不能用同一个标准评价所有内容。破圈内容拼播放，垂类内容拼收藏和申请，产品演示内容拼试用。

---

## 二、五类文案策略

### 1. 大热点破圈型

结构：

```text
大公共热点 / 公共情绪
-> 观点切口
-> 轻点产品
```

目标：

```text
曝光、关注、账号进入更大人群视野。
```

特点：

1. 热点为主，产品为辅。
2. 产品像插播广告一样轻轻点一下。
3. 内容不需要太垂。
4. 可以围绕国事、公共叙事、情感、明星八卦、AI 大趋势等大热点，但必须有合理关联。

适合指标：

```text
播放 / 阅读
点赞
评论
关注
```

风险：

1. 硬蹭。
2. 账号画像跑偏。
3. 只涨播放，不带来有效用户。
4. 公共议题表达不当。

适合使用：

```text
想让更多人知道账号和项目存在。
热点本身足够大。
能通过观点切口自然收回到账号母题。
```

---

### 2. 垂类获客型

结构：

```text
行业热点 / 用户痛点
-> 1-2 层推导
-> 账号母题
-> 产品能力
```

目标：

```text
吸引真实潜在用户靠近。
```

特点：

1. 比破圈型更垂。
2. 重理论、重方法、重场景。
3. 适合讲评论区管理、直播复盘、AI 分析、小商家经营、账号矩阵。

适合指标：

```text
收藏
转发
整体完播
主页访问
申请数
```

风险：

1. 播放量不一定高。
2. 内容容易太干。
3. 需要真实方法论支撑，否则像硬广。

适合使用：

```text
目标用户明确。
产品能力能自然接上。
希望带来申请、下载、试用。
```

---

### 3. 教育信任型

结构：

```text
常见误区
-> 解释机制
-> 给方法
-> 顺手展示工具为什么有用
```

目标：

```text
建立专业可信度。
```

特点：

1. 不一定蹭热点。
2. 适合沉淀账号母题。
3. 让用户觉得“这个人确实懂我的问题”。

适合指标：

```text
收藏
长评论
转发
关注
私信咨询
```

适合内容：

1. 为什么直播复盘不能只看 GMV。
2. 评论区为什么是低成本用户调研。
3. AI 分析评论和 AI 写文案有什么区别。
4. 小团队怎么整理公开反馈。

风险：

1. 太像讲课。
2. 太抽象。
3. 没有案例会显得空。

---

### 4. 产品演示型

结构：

```text
具体场景
-> 打开工具
-> 跑一次
-> 展示结果
-> 告诉谁适合申请
```

目标：

```text
让用户相信工具真的能用，并促成申请 / 下载 / 激活 / 试用。
```

特点：

1. 产品占比高。
2. 播放量可能不高，但转化更直接。
3. 最适合解决用户“这东西到底怎么用”的疑问。

适合指标：

```text
申请数
下载数
激活数
试用反馈
```

适合内容：

1. 用一个直播间评论跑一次复盘。
2. 用一个账号评论看看用户都问什么。
3. 免费学习版申请后怎么用。
4. 一条评论从采集到 AI 分析到 Excel。

风险：

1. 太像广告。
2. 如果演示不顺，会伤信任。
3. 产品 bug 会直接暴露。

---

### 5. 复盘共创型

结构：

```text
用户试用反馈
-> 发现问题
-> 我们怎么改
-> 下一版方向
```

目标：

```text
建立真实感，让早期用户参与产品成长。
```

特点：

1. 后期价值高。
2. 需要真实用户反馈。
3. 能让账号不像冷冰冰工具，更像一个正在长出来的项目。

适合指标：

```text
评论建议
复访
功能反馈
用户共创
```

适合内容：

1. 第一个测试用户把我骂醒了。
2. 这个功能我以为很重要，结果没人用。
3. 为什么免费学习版还要审核。
4. 我们不做自动私信，是因为边界不能乱。

风险：

1. 暴露问题要有分寸。
2. 不能泄露真实客户数据。
3. 不能把未确认方向说成承诺。

## 三、阶段比例建议

早期打 call 阶段：

```text
大热点破圈型：40%
垂类获客型：30%
教育信任型：20%
产品演示型：10%
复盘共创型：0%
```

有真实用户反馈后：

```text
大热点破圈型：30%
垂类获客型：30%
教育信任型：20%
产品演示型：15%
复盘共创型：5%
```

比例不是硬规则，每周根据复盘调整。

## 四、策略选择规则

每个热点或选题都必须先选择策略：

| 判断 | 推荐策略 |
|---|---|
| 热点圈层很大，但和产品只适合轻连接 | 大热点破圈型 |
| 热点和目标用户痛点强相关 | 垂类获客型 |
| 没有强热点，但适合讲清方法和误区 | 教育信任型 |
| 需要证明工具真能用 | 产品演示型 |
| 已有真实试用反馈和改进过程 | 复盘共创型 |

---

## 五、和热点推导链的关系

不同策略使用不同推导层级：

| 策略 | 推荐推导层级 |
|---|---|
| 大热点破圈型 | S -> B -> A -> C -> D，产品轻点 |
| 垂类获客型 | B -> A -> C -> D 或 A -> C -> D |
| 教育信任型 | A -> C -> D |
| 产品演示型 | 场景 -> 产品能力 -> 结果 |
| 复盘共创型 | 用户反馈 -> 产品判断 -> 下一步 |

---

## 六、质检重点

| 策略 | 质检重点 |
|---|---|
| 大热点破圈型 | 是否硬蹭，账号画像是否跑偏，产品是否点到为止 |
| 垂类获客型 | 是否真的打到目标用户痛点，是否有收藏 / 转发价值 |
| 教育信任型 | 是否讲清机制，是否有案例，是否避免空泛说教 |
| 产品演示型 | 是否可信、清晰、能复现，是否避免夸大 |
| 复盘共创型 | 是否保护用户数据，是否真实，是否形成下一步 |

---

## 七、进入文案前必须回答

```text
这条内容属于哪种策略？
它的核心指标是什么？
产品占比应该高、中、低？
是否需要大热点 S 层？
是否需要涛哥确认推导链？
发布后看什么数据判断有效？
```
