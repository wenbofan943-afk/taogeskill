# 热点搜索来源池

> 状态：传播研究规则  
> 主责：定义热点研究去哪里搜、不同来源适合哪个池、适合哪个账号、可信度和使用边界。  
> 边界：本文只指导传播研究和内容规划，不接平台 API，不自动登录，不自动抓取平台后台，不改变客户端、服务器、数据库、积分、license 或发版链路。

---

## 一、核心原则

热点搜索来源池不是网址收藏夹，而是“雷达站点表”。

每轮热点研究必须先读：

```text
账号档案
-> 本来源池
-> 按账号阶段和三池权重选择来源
-> 再搜索热点
```

不要只看一个热榜，也不要只按账号关键词窄搜。

---

## 二、来源字段

每个来源按以下字段记录：

| 字段 | 说明 |
|---|---|
| 来源名称 | 平台、网站、工具或内部资料名称 |
| 来源类型 | 平台热榜 / 搜索趋势 / 行业媒体 / 垂类数据 / 社交监听 / 竞品同行 / 自有反馈 |
| 适合池 | S / B / A-C |

| 适合发现什么 | 这个来源最擅长发现的问题 |
| 可信度 | 高 / 中 / 低 |
| 更新频率 | 实时 / 每日 / 每周 / 不定期 |
| 是否需要登录 | 是 / 否 / 可能 |
| 是否适合自动化 | 是 / 否 / 只适合人工 |
| 风险 | 事实、版权、平台规则、隐私、过度娱乐化等 |
| 备注 | 使用限制和补充说明 |

## 七、每轮来源计划

热点研究开始前，先生成一张来源计划，但不需要让普通用户理解技术细节。

输出格式：

```markdown
## 本轮来源计划

- 账号：
- 账号阶段：
- 时间范围：
- 三池权重：

| 池 | 本轮来源 | 为什么选 | 预计找什么 | 风险 |
|---|---|---|---|---|
| S |  |  |  |  |
| B |  |  |  |  |
| A/C |  |  |  |  |
```

原则：

```text
流量账号：S 池权重大。
转化账号：A/C 池权重大。
信任账号：B + A/C 池权重大。
认知账号：S + B 池权重大。
低频垂类账号：B 池权重大。
```

来源计划由 agent 自动生成；只有来源明显不适合、需要登录、涉及真实数据或有风险时，才停下来问涛哥。

## 八、来源使用边界

1. 不登录用户未授权的平台后台。
2. 不自动发布、评论、私信或互动。
3. 不公开客户聊天、申请表、测试机反馈里的个人信息。
4. 不照抄竞品内容，只拆结构和用户问题。
5. 单来源热点必须标为 C 级，不能直接当事实。
6. 争议性 S 池热点必须做风险过滤，不能硬蹭。
7. S 池热点不能因为不直接相关就立刻淘汰，必须先尝试桥接；桥接失败再淘汰。
