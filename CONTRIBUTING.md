# Contributing

This alpha package is maintained as a document-and-skill workflow.

Before changing behavior:

```text
1. Update the relevant product definition.
2. Confirm the product logic is internally consistent.
3. Compile the change into skill / contract / template files.
4. Update PROJECT_MAP.md and public-manifest.yaml when entrypoints change.
5. Run release-checklist.md and validator scripts before publishing.
```

Do not add real account records, private customer data, platform cookies, tokens, or production runs to the public package.

## Good Contribution Types

```text
sample improvement
skill contract clarification
field dictionary consistency fix
release validation improvement
documentation readability improvement
```

## Must Stay Out Of Scope

```text
platform login
auto publish
auto comment or direct message
credential storage
private account data in examples
```
