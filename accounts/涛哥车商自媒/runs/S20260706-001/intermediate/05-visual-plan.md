## 三、image 提示词

### beat_id：B1

```text
用途：首屏 Hook 画中画
口播对应：价格战这件事，小车商改不了，但客户问过的问题，你不能让它白白流走。
出现时间：0-5s
留存风险：行业反内卷话题太大，观众可能以为空谈。
留人任务：停住划走
画面类型：冲突图 / 对比图
为什么值得生成：把“价格战”和“信任资产”变成一眼能看懂的冲突。
不用这张图会损失什么：前 5 秒停留理由变弱，观众不一定知道这条讲车商现场。
模型 / 质量 / 画幅：gpt-image-2 / high / 9:16
主体：一个车商老板站在门店办公桌前，左侧红色价格标签堆叠下压，右侧手机评论气泡整理成一张复盘表。
场景：二手车门店办公室，玻璃门外有模糊车影，桌上有手机、钥匙、便签和打印表。
动作：人物低头看手机，评论气泡从手机浮出，逐渐排列成整齐表格。
镜头 / 构图：竖屏 9:16，中景，人物在右三分之一，左侧留出冲突空间，上方留字幕安全区。
光线 / 情绪：门店冷白灯 + 手机屏幕光，真实经营压力感。
字幕安全区：顶部 20% 不放关键细节，主体避开下方字幕区。
中文提示词：现实主义商业纪录片风格，二手车门店办公室，一个车商老板看着手机，左侧红色价格标签层层下压象征价格战，右侧客户评论气泡逐渐整理成一张“问题复盘表”，玻璃门外有模糊车辆，桌上有车钥匙、便签和打印表，竖屏 9:16，中景构图，人物在右三分之一，顶部留字幕安全区，手机屏幕光和门店冷白灯，真实、不夸张、适合短视频口播画中画。
五槽提示词：
Scene: 二手车门店办公室，玻璃门外有模糊车影，冷白灯和手机屏幕光。
Subject: 车商老板低头看手机，面对价格标签和客户评论问题。
Important Details: 左侧红色价格标签堆叠下压，右侧评论气泡整理成复盘表，桌上车钥匙、便签、打印表，竖屏构图，上方字幕安全区。
Use Case: 短视频口播首屏画中画，用于停住划走动作。
Constraints: 不出现真实平台界面、手机号、微信号、车牌、真实用户头像；不要夸张科幻，不要暗示截流或自动私信。
负面提示：no phone numbers, no WeChat IDs, no license plates, no real user avatars, no watermark, no exaggerated sci-fi, no spam marketing, no auto-DM implication
是否允许文字：允许极短中文“价格战 / 信任战”，不允许额外文字。
剪辑建议：首屏出现 2.5 秒，配合 Hook 前半句；评论气泡可用后期轻微动效。
风险备注：如果文字不清，剪辑层后加字幕，不强求模型画准。
验收标准：3 秒内能看出价格战压力和客户问题沉淀的对比。
```

### beat_id：B3

```text
用途：客户问题现场图
口播对应：他是不是一直问车况？是不是担心调表？是不是纠结质保？
出现时间：20-35s
留存风险：连续列举问题，容易变成干口播。
留人任务：补信息
画面类型：场景图 / 证据图
为什么值得生成：让车商一眼看到“这就是我评论区每天发生的事”。
不用这张图会损失什么：现场感不足，观点像泛泛建议。
模型 / 质量 / 画幅：gpt-image-2 / high / 9:16
主体：手机屏幕特写，直播间公开评论问题以脱敏气泡形式出现。
场景：车商直播桌面，有补光灯边缘、车钥匙、库存车资料夹。
动作：评论气泡围绕手机浮出，短句包括“车况怎么样”“有事故吗”“能贷款吗”“质保多久”。
镜头 / 构图：俯拍近景，手机在画面中心偏下，评论气泡向上展开，顶部留字幕安全区。
光线 / 情绪：直播间屏幕光，真实忙碌。
字幕安全区：顶部留空，气泡不要压住主体。
中文提示词：二手车商直播桌面俯拍近景，手机屏幕显示脱敏的公开评论气泡，周围有补光灯边缘、车钥匙、库存车资料夹，评论气泡短句包括“车况怎么样”“有事故吗”“能贷款吗”“质保多久”，画面真实、干净、小屏可读，竖屏 9:16，顶部留字幕安全区，不出现真实头像和个人信息。
五槽提示词：
Scene: 车商直播桌面，补光灯、车钥匙、库存资料夹。
Subject: 手机屏幕和浮出的公开评论气泡。
Important Details: 评论短句清晰但脱敏，不出现真实用户头像，俯拍近景，手机中心偏下，顶部留白。
Use Case: 短视频口播画中画，用于增强客户问题现场感。
Constraints: 不出现手机号、微信号、真实账号名、真实用户头像、车牌；不要像平台后台抓取工具。
负面提示：no private data, no usernames, no phone numbers, no WeChat IDs, no license plates, no watermark, no surveillance UI
是否允许文字：允许短中文评论，必须清晰且只出现指定短句。
剪辑建议：列举客户问题时逐个放大气泡。
风险备注：文字若生成不稳，使用无文字气泡 + 剪辑字幕替代。
验收标准：小屏一眼能看出客户在反复问车况、事故、贷款和质保。
```

## 四、剪辑使用建议

1. 首屏 B1 必须优先使用，前 0-5 秒解决停留问题。
2. B3 和 B6 是可信度核心，一个讲客户现场，一个讲工具边界。
3. B4 / B5 二选一即可，避免整条视频画中画过密。
4. CTA 段不建议生成复杂画面，保留口播和字幕即可。
